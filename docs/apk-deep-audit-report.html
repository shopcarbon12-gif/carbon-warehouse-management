4. Security, OTA, and settings

  • Device ID — HandheldDeviceIdentity uses Android ID for deviceId on APIs (matches server devices.android_id design).
  • Boot sequence — Auto-detect Chainway manufacturer → Chainway scanner, else Zebra path (rfid_manager.dart).
  • Power slider — Dashboard header slider 0–300; persists via MobileSettingsRepository.setGlobalAntennaPower; reapplied to scanner. Native layer currently logs power in debug; verify it drives API3 / Chainway reflection.
  • OTA dialog — On boot, update dialog is dismiss-only (no in-dialog install). Download & install lives in dashboard header — acceptable UX but differs from “single dialog does everything” specs.
  • OTA polling — Dashboard re-polls fetchMobileStatus every 3 minutes; snackbar when update available.
  • Release signing — build.gradle.kts uses signingConfig = debug for release — fine for internal QA; not Play Store ready.

6. Hardware & Geiger (RSSI) fidelity

  • Zebra — CarbonZebraRfidController.kt uses API3 tag RSSI (getPeakRSSI() etc.) when linked — suitable for Geiger proximity.
  • Chainway — CarbonChainwayRfidController.emitEpc sends hardcoded rssi: -55 for every tag. Geiger mode will not reflect real distance on Chainway until RSSI is read from the device API.
  • Stubs — If native bridge fails to link, scanners still “connect” but won’t emit real reads; UI relies on SIMULATE buttons for QA.

7. Ghost filter & edge streaming

  • Server-driven visibility — EPCs queue to postEpcVisibility; invisible tags never enter session or edge batch (matches Clean 10 “ghost” intent).
  • Geiger exception — GEIGER_FIND bypasses ghost queue and edge ingest by design (rfid_manager.dart).
  • Docs drift — Comment on postEpcVisibility still names legacy Senitron flags; server uses is_visible_to_scanner semantics.

8. Dependencies & native bundles

  • Flutter: http, provider, shared_preferences, mobile_scanner, audioplayers, install_plugin, android_id, device_info_plus, package_info_plus, google_fonts, lucide_icons.
  • Android: Zebra API3_LIB-release-2.0.2.82.aar in app/libs; optional Chainway JARs via fileTree.
  • Path packages carbon_chainway_driver / carbon_zebra_driver are placeholders; real glue is in app Kotlin + Dart RfidVendorChannel.

  3. Encode suite — Wire to live catalog APIs and server commissioning endpoints; remove stub rows; real ZPL gateway if printers are network-managed.
  4. Status change — Align buckets with Clean 10 vocabulary and/or call bulk-status API with server validation.
  5. Clean bin mobile — Thin screen calling web’s clean-bin endpoint with bin scan.
