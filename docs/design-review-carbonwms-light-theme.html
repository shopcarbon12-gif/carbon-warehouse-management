CarbonWMS light theme — unified direction

This document merges your scheme.png palette with DESIGN (2).md (“Industrial Precision / Aggressive Functionalism”): tonal surfaces instead of noisy outlines, teal as the single accent, strong typographic hierarchy, and monospace for SKU/EPC data. It also addresses current UI issues: washed-out page titles, mixed dark panels on light pages, and controls that fail contrast.

Primary #0B9597, Secondary #587E7E, Tertiary #031A84, Neutral #737878. Typography: headline / body / label levels. Buttons: primary, secondary, inverted, outlined; light search field.

Typography scale (proposal)

Default UI (current “large”)Base ~17–18px (1.0625–1.125rem); comfortable line height 1.5–1.55.
“Expanded” tier+1 step (~19–20px root or component multiplier) for accessibility / warehouse use; sidebar width and row min-heights grow with it.

Principles excerpt — DESIGN (2).md

  • No-line architecture: structure from surface tiers (#FFF / #F3F3F4 / #EEE), not busy borders.
  • Dual type: Manrope/Inter for UI; monospace for codes.
  • 56dp targets for primary actions in field-heavy flows where feasible on web.
