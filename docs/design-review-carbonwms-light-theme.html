Design System Strategy: Industrial Precision D:\Downloads\DESIGN (2).md

1. Overview & Creative North Star

The North Star for this design system is “Aggressive Functionalism.”

In an industrial warehouse environment, there is no room for decoration—only performance. This system rejects the “web-style” template in favor of an editorial, high-precision aesthetic. We move beyond standard UI by utilizing a “No-Line” architecture, where structure is defined by tonal weight and negative space rather than strokes.

The layout should feel like a high-end technical manual: asymmetric balance, bold monochromatic shifts, and a ruthless commitment to touch ergonomics. We are not just building an app; we are building a digital tool that feels as solid and reliable as the hardware it runs on.

2. Colors & Tonal Architecture

The palette is rooted in a pure white base, contrasted by a deep, authoritative Teal and a spectrum of functional grays.

The “No-Line” Rule

Explicit Instruction: Prohibit the use of 1px solid borders for sectioning or containment.

  • Separation: Use background shifts (e.g. surface-container-low #F3F3F4) against the pure white surface-container-lowest (#FFFFFF).
  • Definition: Boundaries are created through the surface hierarchy. A card should never be “outlined”; it should “exist” as a tonal change.

Surface Hierarchy & Nesting

Treat the UI as a series of physical plates. Use the following tiers to define depth:

  • Base: surface-container-lowest (#FFFFFF) for the primary canvas.
  • Sectioning: surface-container-low (#F3F3F4) for grouping related inputs.
  • Interaction: surface-container (#EEEEEE) for inactive buttons or secondary fields.
  • Nesting: Place a high-priority data read-out (using surface-bright) inside a surface-container-high zone to create a “recessed” look that draws the eye.

Signature Textures

  • The Teal Gradient: For primary actions (Login, Submit), transition from primary to primary_container. This subtle vertical gradient prevents the “flat” look and adds a premium, tactile weight.
  • Glassmorphism: Use for floating notifications or “Scan Success” overlays. Use surface colors at 80% opacity with a 20px backdrop-blur to maintain context of the warehouse floor while focusing the user.

CarbonWMS: Toolbar actions follow the Inventory catalog pattern — solid green / blue / slate / orange, white text, no teal gradient on those controls. Teal remains for brand accent, links, and header strip. Glassmorphism only where we already use overlays.

3. Typography: The Dual-Engine System

We utilize two distinct type families to separate “UI Instruction” from “Industrial Data.”

  • UI Engine (Manrope/Inter): High-readability sans-serif for navigation and labels.
    • Display/Headline: Use Manrope for an aggressive, geometric feel.
    • Body/Title: Use Inter for neutral, utilitarian communication.
  • Data Engine (Space Grotesk / Monospace): All EPC, UPC, SKU, and Location data must use the label-md or label-sm scales (Space Grotesk). The fixed width ensures that alphanumeric codes are easily scannable and never misread.

Editorial Hierarchy: Use a “High-Contrast” scale. If a headline is headline-lg, the supporting label should be label-sm. This dramatic difference in scale eliminates ambiguity in high-stress environments.

CarbonWMS: Keep existing app font families (no switch to Manrope/Inter/Space Grotesk as new dependencies). Apply the same idea: clear hierarchy via size/weight; monospace for codes if already in use.

4. Elevation & Depth

Depth is achieved through Tonal Layering, not structural lines.

  • Layering Principle: Stack surfaces. A surface-container-lowest card sitting on a surface-container-low background provides sufficient contrast for the human eye without the visual “noise” of a border.
  • Ambient Shadows: For floating elements (Modals/Quick-Actions), use extra-diffused shadows.
    • Specs: Y: 8px, Blur: 24px, Color: on_surface @ 6%. This mimics soft, overhead warehouse lighting.
  • The “Ghost Border” Fallback: If a high-contrast scenario (like an error state) requires containment, use outline_variant (#C6C5D4) at 20% opacity. Never use 100% opaque borders.

5. Components & Ergonomics

The 56dp Glove-Touch Rule

Every interactive element must have a minimum hit area of 56dp. This accounts for industrial glove usage and rapid, high-vibration movement.

  • Buttons:
    • Primary: Teal gradient, rounded-md (0.375rem), white text.
    • Secondary: surface-container-high background, on_surface text. No border.
  • Input Fields: Use surface-container-low backgrounds. Labels should be label-md and always visible (never floating placeholders). Height must be exactly 64dp to accommodate large touch targets.
  • Data Cards: Use 1.5 (0.5rem) or 2 (0.7rem) spacing between elements. Forbid divider lines; use a 3 (1rem) vertical gap to separate SKU entries.
  • Inventory Chips: Use rounded-sm (0.125rem) for a more “mechanical” look. Use tertiary_container for high-priority alerts (e.g. Low Stock).

CarbonWMS: Aim for generous tap targets on web where practical; catalog toolbar buttons stay solid role colors (not necessarily teal gradient primary). Inputs: visible labels + contrast fixes in light theme.

6. Do’s and Don’ts

Do

  • DO use Monospaced fonts for all UPC and SKU data to ensure characters like ‘0’ and ‘O’ are distinguishable.
  • DO rely on the Spacing Scale (specifically spacing-4 and spacing-6) to create “breathing room” that guides the eye.
  • DO use surface-dim for inactive states to maintain the “No-Line” aesthetic.
  • DO treat the UI as an editorial layout—use bold, large headings to anchor the screen.

Don’t

  • DON’T use 1px borders to separate list items. Use a 1px surface-container-highest background strip if separation is critical, or simply use spacing-px.
  • DON’T use standard blue (#0000FF). Stick strictly to the primary Teal for a premium feel.
  • DON’T use “Soft” rounded corners (e.g. xl). Keep the industrial vibe with sm or md (0.125rem to 0.375rem).
  • DON’T use drop shadows on buttons; let the tonal shifts and gradients provide the affordance.

CarbonWMS: Import in the catalog intentionally uses blue as a semantic role color — we keep that. Avoid neon #0000FF; use the same controlled blue token as the catalog. Shadows on buttons: prefer flat solids; optional very soft shadow only if needed for contrast on light backgrounds.
