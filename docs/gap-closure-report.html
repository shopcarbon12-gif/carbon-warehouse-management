CarbonWMS — Gap closure report

This document records work performed to align the Next.js web app and the Flutter handheld APK with the findings in docs/web-deep-audit-report.html, docs/apk-deep-audit-report.html, and the chronological prompt audit. No intentional reverts of features landed after those baselines.

Scope & outcome
  • Closed API authentication for mobile (Bearer + cookie), JSON 401 on /api/* without session (no HTML redirect to login).
  • Closed Transfer slips: receive / missing outcomes, manager-scoped list/detail, append EPCs; dashboard RBAC for warehouse floor on /inventory/transfers.
  • Closed Mobile endpoints: clean bin by code, item detail by EPC; Flutter screens and WmsApiClient methods.
  • Closed Broad codemod: protected API routes use getSessionFromRequest(req) so Bearer tokens work consistently.
  • Closed Handheld: bulk status, RFID catalog search + commission wiring, inventory lookup bin enrichment, Chainway RSSI via reflection on tag reads.
  • Encode suite Search + Scan & Print call live catalog-search and postRfidCommission (server ZPL + printer). Upload tab persists failed WRITE jobs (commission_retry_queue.dart) and retries via RETRY ALL.

1. Web (Next.js)

1.1 proxy.ts

  • All /api/* routes not on the public allowlist now require a valid session from either Authorization: Bearer <jwt> or the wms_session cookie.
  • Missing auth returns 401 with JSON { "error": "Unauthorized" } — never a redirect to /login (fixes mobile fetch parsing and tooling).
  • Page routes keep the previous cookie-only flow: unauthenticated users redirect to /login; RBAC for admin-only and warehouse-floor paths unchanged except where noted below.
  • Public allowlist unchanged for health, auth, webhooks, handheld edge paths, and other entries already listed in file.

1.2 RBAC — lib/auth/dashboard-rbac.ts

  • /inventory/transfers is allowed for warehouse floor roles (not admin-only), matching operational use of transfer slips in the warehouse.

1.3 Transfer slips — lib/queries/transfer-slips.ts

  • Added setTransferItemsOutcome (and related query support) to mark slip line items as received or missing for IN/receive flows.

1.4 API routes

PathChange
GET/POST …/api/inventory/transfer-slips Session via getSessionFromRequest; listing/creation scoped to MANAGER (aligned with floor managers, not only ADMIN).
GET/PATCH …/api/inventory/transfer-slips/[slipNumber] Same session helper; POST body supports discriminated actions: append_epcs, receive (with per-line outcome).
POST …/api/mobile/clean-bin New. Body { "binCode": string }; MANAGER; resolves bin in an active location and clears contents per server rules.
GET …/api/mobile/item-detail?epc= New. MANAGER or WAREHOUSE_OPS (membership scopes); returns item/bin detail for a 24-hex EPC.
app/api/**/route.ts (wide) Migrated from getSession() to getSessionFromRequest(request) so the same handlers accept Bearer tokens from the handheld app.
app/api/settings/status-labels/route.ts Admin paths use requireAdmin(req) with the incoming Request for Bearer-aware checks.

1.5 Tests — e2e/smoke.spec.ts

  • Added assertion that GET /api/dashboard/summary without credentials returns 401 and JSON with an error field.
  • Full Playwright smoke (login + health) requires a healthy dev server with a valid .next build; a broken or partial .next/dev tree can yield 500 unrelated to these changes.

1.6 Verification

  • npx tsc --noEmit — clean at time of report generation.

2.3 Android — Chainway RFID

  • CarbonChainwayRfidController.kt: RSSI read via reflection on vendor tag objects; emitEpc(hex, rssi?) with a safe default when RSSI is unknown (e.g. -55).

2.4 Verification

  • flutter analyze — no errors; remaining messages are info level in various screens (const, deprecated dropdown API in status screen, etc.).
  • Release APK: built via mobile/carbon_wms/scripts/build-apk.ps1. On Windows, run from a path without spaces (e.g. junction D:\cwm\mobile\carbon_wms) so Dart native-asset hooks do not fail. Artifact: build/app/outputs/flutter-apk/app-release.apk (mirrors the repo path via the junction).

3. Repository hygiene

  • .gitignore: ignore Playwright /test-results/ and /playwright-report/ so local e2e runs do not dirty commits.
  • One-off codemod script for API session migration was removed after use (not part of long-term tooling).

4. Deploy & operations

  • Production rollout: push to main and trigger Coolify Redeploy (rebuild) if Docker or app entrypoints change; npm run deploy:coolify may be used when COOLIFY_DEPLOY_WEBHOOK_URL (and optional API token) are configured locally.
  • OTA APK: build release APK, then npm run upload:mobile-apk with WMS_APP_PUBLIC_BASE_URL and WMS_UPLOAD_SESSION_COOKIE (admin session), or use Settings → Mobile updates in the web UI.
  • GET /api/health remains DB-free for container health; readiness stays on /api/health/ready if used.
