CarbonWMS — Deep web audit (Next.js App Router)

Static review of the desktop/admin app under app/, lib/, components/, and production tooling. No production smoke run is implied.

Stack snapshot
  • Framework: Next.js 16.2.1, React 19.2.4, TypeScript, Tailwind v4 (package.json).
  • Data: PostgreSQL via pg pool (lib/db.ts) — no Prisma/Drizzle.
  • Auth: JWT session in cookie wms_session; mobile uses Authorization: Bearer on API routes (lib/get-session-from-request.ts).
  • Output: output: "standalone" for Docker (next.config.ts).
  • UI shell: app/(main)/layout.tsx → WmsShell + sidebar; root app/layout.tsx fonts + dark html.

2. Health, config & production image

  • Liveness — GET /api/health returns JSON quickly, no DB (app/api/health/route.ts).
  • Readiness — GET /api/health/ready runs SELECT 1 when DATABASE_URL is set (app/api/health/ready/route.ts).
  • Dev HSTS — Strict-Transport-Security: max-age=0 only in development (next.config.ts).
  • Docker — Multi-stage build, standalone output, non-root nextjs, writable .next/cache, HEALTHCHECK → /api/health, entrypoint migrations (Dockerfile, docker-entrypoint.sh).
  • Base path — Optional WMS_BASE_PATH for subpath deploy + redirect from / (next.config.ts).

5. Data & migrations

  • Runner: npm run db:migrate → scripts/migrate-incremental.ts → shared migrator applies scripts/schema.sql baseline + ordered files in scripts/migrations/ (20 SQL files including RFID matrix, tenant settings, reports, enterprise v2, Clean 10 statuses, Shopify removal, etc.).
  • Seed: npm run db:seed → scripts/seed.ts.
  • Types: types/db.ts and query modules under lib/queries/ align with evolving schema.
  • Ordering — Two files share prefix 009_*.sql; ensure deployment order is correct on greenfield DBs.

6. Background worker

npm run worker — scripts/worker.ts claims sync_jobs rows; processes lightspeed_catalog / lightspeed_pull via executeLightspeedCatalogJob; other job types currently stubbed with sleep + completed update. Separate process from Next — must run in production if queued catalog sync should complete without hitting an API trigger path.

  • Status system: Clean 10 architecture — 019_clean_10_status_architecture.sql, enforcement in lib/server/status-label-enforcement.ts, tooltips, statuses workspace with read-only rule summary.
