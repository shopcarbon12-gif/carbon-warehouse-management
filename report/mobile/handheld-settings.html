Handheld Settings

Device-level settings for the handheld. Accessible from the app bar or settings icon. Includes security settings (biometric toggle), OTA update controls, server connection information, and device identity details. These settings affect only the current physical device, not the tenant configuration.

Security Settings

Biometric Login Toggle
Toggle Switch
Enables or disables biometric authentication (fingerprint/face) for this device. When turned off, the user must always enter their email and password to log in. When turned on, biometric credentials are saved in the device's secure vault. Turning off clears the saved biometric credentials.
Suggestion: When turning off biometric login, confirm with the user: "Disable fingerprint login? You will need to enter your password each time." Also, when turning on, explain the security implication: "Anyone who can unlock this device will be able to log in as you." This is important for shared devices — if another staff member's fingerprint is registered on the device's Android lock screen, they could gain access to this user's WMS account.

Update Settings

Check for Update Button
Button
Manually triggers a check for a new app version from the server's OTA update API. Shows the current app version and, after checking, shows whether an update is available. If an update is available, an install prompt appears. This is the same check that the dashboard runs automatically every 3 minutes.
Suggestion: After checking, always display the result — "Up to date: v1.1.1+28" or "Update available: v1.1.2+29 — tap to install." Don't just show a loading spinner and then nothing if there's no update available. Also, show the last check timestamp: "Last checked: 4 minutes ago."
Current App Version Display
Info Display
Shows the currently installed version of the mobile app. Format: version number from pubspec.yaml (e.g., "v1.1.1+28" where 1.1.1 is the version name and 28 is the build number).
Suggestion: The version display is important for support and diagnosis. Make it easy to copy — a long-press or "Copy" button next to the version number lets staff quickly share it with support staff when reporting an issue.

Device Information

Server URL Display
Locked Display
Shows the server address the app is connected to. The URL is read-only and cannot be changed by the user — it is compiled into the app. This is shown so support staff can verify the device is connected to the right server.
Suggestion: Replace the raw server URL with a user-friendly server name, and move the technical URL behind a "Show details" tap. Most staff don't need to know the server address; support staff who do can tap to reveal it. This also reduces the chance of staff accidentally thinking the URL is something they need to change.
Device Identity Info
Info Display
Shows the device's Android ID, MAC address, serial number, and other hardware identifiers. These are the same values displayed on the web Device Binding page. This information helps an admin identify and authorize the device.
Suggestion: Add a "Share device ID" button that copies all the device identity information to the clipboard in a formatted, easy-to-paste format: "Device ID: [MAC] / [Android ID]". This makes it much easier for a staff member to send their device info to an admin over a messaging app without reading out long hex strings verbally.

Overall Handheld Settings Suggestions

1. No logout from settings: The sign-out button is on the dashboard, not in settings. If a user navigates to settings expecting to find sign-out options, they won't find it. Consider adding a sign-out option in the settings screen as well.

2. No data clear option: If a device has cached data or a corrupted local state, there is no way to clear it from settings. A "Clear app data / Force resync" option would help support staff troubleshoot devices without requiring a full app reinstall.

3. Settings changes should not require a restart: If any setting change (like biometric toggle) requires an app restart to take effect, this should be explicitly stated. Silently broken settings are frustrating.
