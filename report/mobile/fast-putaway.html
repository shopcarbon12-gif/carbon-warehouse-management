📱 Mobile App

Fast Bin Putaway

A two-phase workflow for quickly assigning RFID-tagged items to a specific bin. Phase 1: scan the item tag. Phase 2: scan the bin's barcode. The item is then recorded as belonging to that bin. Designed for speed — staff can process many items rapidly. Also supports "color scope" — assigning a specific color attribute to a bin assignment.

Phase 1 — Scan Item

Item Scan (RFID)
RFID Scan
The screen starts in item-scan mode. The RFID reader waits for a tag to be read. When a valid item tag is scanned, the item's details (name, SKU, current bin) are displayed briefly and the screen transitions to Phase 2 (bin scan). If the tag is not recognized, an error is shown.
Suggestion: Phase transitions should be very obvious on a small screen. After scanning an item, use a large clear visual indicator: "Item scanned: [Product Name] — Now scan the bin." Staff moving quickly may miss subtle state changes. Also, allow canceling a scanned item and returning to Phase 1 without restarting the entire workflow.

Phase 2 — Scan Bin

Bin Scan (Barcode)
Barcode Scan
After an item is scanned, the screen switches to bin-scan mode. The staff member scans the barcode label on the storage bin. The item is then assigned to that bin in the system. After a successful assignment, the screen returns to Phase 1, ready for the next item.
Suggestion: The transition back to Phase 1 after a successful assignment should include a visible counter: "12 items putaway this session." This gives staff a running total and confirms each assignment worked. Also, add a short delay or "undo last" option (5-second window) in case a staff member scans the wrong bin — mis-assignments are easy to make when working fast.
Color Scope Bottom Sheet
Modal
A bottom sheet (slide-up panel) that can be opened to assign a "color scope" to a bin assignment. The color scope is a business-specific attribute that associates the item with a color category when placed in that bin. This is relevant for inventory organization by color at certain locations.
Suggestion: The "color scope" feature may be confusing to staff who don't know what it means. Rename it to something more descriptive of the actual business use case, or add a brief explanation on the bottom sheet: "Color scope assigns a color category to items in this bin, used for inventory organization." Also, make it clear whether the color scope is optional or required for certain workflows — if it is always optional, say so.

Bottom Action Bar

Tactical Bottom Bar Buttons
Large Bottom Bar Buttons
The bottom of the screen has large, full-width buttons (designed for use with gloves) built with the TacticalEmeraldButton and TacticalSlateButton components. The green (Emerald) button is the primary action; the grey (Slate) button is secondary or cancel. Both are at least 80px tall for easy tapping.
Suggestion: The large bottom bar buttons are an excellent design decision for warehouse floor use. Ensure button labels clearly describe the action (not just "Confirm" but "Save bin assignment") so staff know exactly what will happen. Also, the green/grey color coding is intuitive: green = proceed, grey = go back — maintain this consistently across all screens.

Overall Fast Putaway Suggestions

1. No session summary: When a staff member finishes a putaway session and goes back to the dashboard, there is no summary of how many items were put away or a log of the assignments made. A session summary screen would help with verification and catching mistakes.

2. No "already in this bin" detection: If a staff member scans an item that is already in the target bin, the system should alert them: "This item is already in bin A3 — no change needed." This prevents unnecessary re-assignments and helps catch accidental duplicate scans.

3. Multi-item scan before bin: Consider an "advanced mode" where staff can scan multiple items before scanning the bin — useful when placing a whole armful of items into the same bin at once. The current one-item-one-bin sequential flow is fast for singles but slow for batch placements.
