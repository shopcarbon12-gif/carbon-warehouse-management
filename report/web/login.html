Login

Entry point for all web users. A centered form with email, password, and a sign-in button. No logo or product name is shown on the page. After login, the user is sent to the dashboard.

Recommendations

Login Form

Email Field
Text Input
Standard email address input. Browser auto-fill works. The keyboard on mobile devices opens in email mode (with @ key visible).
Email keyboard mode on mobile is a small but correct touch — easier to type addresses without switching keyboard layouts.
Password Field
Password Input
Text is hidden as the user types. No show/hide eye icon is present. No "Forgot password?" link appears near it.
No way to reveal the password while typing, which is frustrating when a mistake is made — the user must delete and retype the whole password. No password recovery path exists anywhere on this screen.
Add a show/hide eye icon inside the field — standard on every modern login form. Add a "Forgot password?" link below the field. Even if full reset isn't built yet, link to an admin contact or a placeholder page so users are not left stuck.
Sign In Button
Primary Button
Submits the form. Goes into a loading state while the server responds. On success, redirects to the dashboard. On failure, shows an error message below the form.
The button disables itself and shows a pending state during the login request — prevents double-clicks and gives feedback that the action was received.
The error message on failure uses developer-style language that warehouse staff will find confusing.
Use plain language: "That email or password is incorrect — please try again." Never show raw error codes or technical messages to end users.
Page Identity (Branding)
Layout
The login page shows only the form — no logo, no product name, no company name anywhere on the page.
A blank login form with no branding means a user can't confirm they're on the right system. If someone accidentally navigates to staging instead of production, there is no visual difference. New employees don't know what they are logging into.
Add the CarbonWMS logo and name above the form. If multiple environments exist (production, staging, dev), show the environment name clearly — a small "STAGING" banner is enough to prevent confusion.

Overall Assessment

No password recovery is the biggest gap here. Every locked-out staff member requires an admin to manually reset their account. This is a support bottleneck that will grow as the team size grows. It should be the first thing added to this page.
Priority fixes:
1. Forgot password link or flow.
2. Show/hide password toggle.
3. Add logo/branding above the form.
