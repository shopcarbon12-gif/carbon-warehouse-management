RFID EPC General Settings

Controls the global encoding rules used when generating RFID tags for this warehouse. Every tag written by the system follows these rules. Getting this wrong means tags won't be read correctly by scanners or won't match the expected item data. This is an advanced configuration page.

Encoding Settings Form

Encoding Standard Dropdown
Dropdown
A dropdown with two options: "SENITRON (default)" and "Custom." SENITRON is the encoding format used by Senitron RFID hardware. Choosing SENITRON means tags are generated according to Senitron's specific bit layout. Choosing Custom allows using a different layout defined in the EPC Profiles page. This setting applies to all tag generation for this tenant.
Suggestion: The terms "SENITRON" and "Custom" will be meaningless to most users. Add a brief explanation below the dropdown: "SENITRON (default): Use this if your RFID readers and printers are Senitron hardware. Custom: Use this if you have a different encoding layout — configure it in EPC Profiles below." Also, changing this setting after tags have already been printed could break scanning for existing tags. Add a warning: "Changing the encoding standard will affect all future tag printing. Existing printed tags will not be updated."
Company Prefix Field
Text Input
A text field for entering the company's hexadecimal prefix — a unique identifier embedded in every tag generated. The field only accepts uppercase hex characters (0–9, A–F) up to 12 characters long. The input is automatically uppercased as you type. If left empty when saving, it defaults to "F0A0B".
Suggestion: The word "hexadecimal" and the concept of a company prefix are very technical. Add a tooltip or help text: "Your company's unique identifier embedded in every RFID tag. This was set during initial setup. Do not change it unless you are re-configuring your entire tag system — doing so will make all existing tags unreadable." The default "F0A0B" fallback is a risk — if a user accidentally clears the field and saves, their prefix silently changes to the default. Add a confirmation or at minimum a warning when saving with the default value.
Active EPC Profile Dropdown
Dropdown
An optional dropdown showing all configured EPC profiles. Selecting one makes it the "active" profile — the default layout used when encoding tags. Can also be set to "— None —" meaning no specific profile is enforced. The dropdown is populated from the EPC Profiles list (the sub-page for configuring bit layouts).
Suggestion: When "None" is selected, what layout is actually used? This is unclear. There should be a note explaining what happens when no profile is active: "If no profile is selected, the system will use the SENITRON default bit layout." Also, profile names in the dropdown may not be meaningful — include a brief description of each profile's encoding scheme (e.g., "Store A — 40-bit item / 36-bit serial").
Save / Reset Buttons
Primary/Secondary Buttons
Save applies the changes and shows either "Saved." (success) or an error message. Reset discards unsaved changes and returns the fields to the last saved values. Both buttons are disabled while a save is in progress.
Suggestion: The "Saved." message disappears quickly and is easy to miss. Make success messages more visible and persistent (at least 5 seconds). Also, since this is a high-impact setting, consider requiring a typed confirmation (like typing "CONFIRM") before saving — or at minimum a clear warning dialog: "Changing EPC settings affects all tag generation. Are you sure?"

Overall EPC General Settings Suggestions

1. No audit trail: Changes to the encoding standard or company prefix should be logged in the activity audit with the old and new values. This is critical for diagnosing tag generation issues after a configuration change.

2. Impact warning missing: This is one of the highest-impact settings in the entire application. Changing the company prefix or encoding standard without understanding the implications can break all tag scanning. The page needs a visible warning banner at the top, not just field-level help text.

3. Relationship to EPC Profiles is unclear: This page references EPC Profiles but does not link to them. Adding a direct link to "Manage EPC Profiles" from this page would help admins navigate the configuration flow.
