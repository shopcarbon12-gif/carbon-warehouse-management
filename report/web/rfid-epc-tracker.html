EPC Tracker

Look up any RFID tag or item by EPC code, SKU, or keyword. Shows a detail panel with the item's current status, bin, and a full history timeline of every event on that tag — scans, status changes, location moves. Minimum 2 characters to trigger search, 320ms debounce.

Search

Live Search Input
Text Input (debounced)
Accepts any text query — EPC code (exact match), SKU, or keyword. Requires at least 2 characters. Results appear 320ms after typing stops. Direct EPC match jumps straight to that tag's detail view. Keyword search shows a results list to choose from.
The two-mode behavior (exact EPC → direct detail, keyword → list) is smart — an admin who scans an EPC directly gets to the right place instantly. The 2-char minimum and debounce prevent pointless server calls.
No placeholder text tells users what formats are accepted. Entering a full EPC code vs a product name vs a partial SKU all produce different result types, but there is no hint of this in the UI.
Add a placeholder: "Search by EPC code, SKU, or item name." Consider a format toggle or a visual badge showing "EPC match" vs "Keyword search" mode when results are shown, so users understand why the result looks different.

Item Detail Panel

Item Detail Panel
Detail View
Shows all details for a selected tag: EPC code, associated product name/SKU, current status, current bin assignment, and the commissioning date. Displays prominently when an EPC is found.
Showing all relevant item details in one panel — without navigating away — makes this the fastest diagnostic tool in the app for "where is this specific item right now?"
The current status is displayed as plain text with no color coding. A "stolen" item looks the same as an "in-stock" item visually. This slows down the reader's eyes during a busy investigation.
Apply status color coding: green for in-stock, red for stolen/damaged/tag_killed, amber for return/pending states. The status should be the most visually prominent piece of information in the detail panel.
History Timeline
Event Log
A chronological list of every event associated with the tag: each scan reading, status change, bin assignment change, and commissioning event. Each entry shows timestamp, event type, and any associated metadata.
Full tag history in a timeline format is exactly what's needed for investigating a lost or disputed item. Being able to see "this tag was last scanned 3 days ago at Bin A3, then nothing" answers questions that previously required checking raw database tables.
The timeline is flat — all event types look the same. A routine scan and a status change are displayed identically, making it hard to spot the important events quickly when a tag has hundreds of scan entries.
Use visual differentiation by event type: status changes in amber, location moves in blue, scans in grey, commissioning in teal. Allow filtering the timeline to show only "status changes" or "location changes" to cut through the noise of routine scans.

Overall Assessment

The EPC Tracker is one of the most useful diagnostic tools in the app. The ability to look up any tag and see its complete history in one place directly supports the core warehouse management use case: knowing exactly where a specific item is and what happened to it.
Top improvements:
1. Color-code item status in the detail panel — make "stolen" visually alarming, "in-stock" visually reassuring.
2. Color-code timeline events by type — separate scans from status changes visually.
3. Add search placeholder text.
