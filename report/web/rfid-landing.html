RFID Workflows — Landing Page

A simple hub page with links to three areas: an external legacy tag encoding portal (external URL), the Inventory Catalog, and the Handheld API developer docs. No tools or data of its own — purely a navigation page.

Navigation Links

Link to External Legacy Encode Portal
External Link
Opens an external URL for an older tag encoding system. Opens in a new tab. No description of what the external portal does or why a user would need it.
A plain link to an external system with no explanation is confusing. Users don't know whether they should use the external portal or the built-in Commissioning page. Having both options creates uncertainty about which is correct.
Add a brief description: "External legacy encoder — use only for [specific use case]. For new tag printing, use the Print / Commission page instead." If the legacy portal is being phased out, add a deprecation notice and a timeline for removal.
Page Purpose
Page Layout
This page exists as a hub/landing for RFID features. However, all the features it links to (Inventory, Commissioning, Cycle Counts) are already accessible directly from the sidebar navigation.
A landing page that only links to things already in the sidebar navigation adds a step without adding value. The RFID section in the sidebar already organizes all RFID features — a separate landing page is redundant.
Consider removing this landing page entirely and relying on the sidebar accordion for RFID navigation. Or repurpose it as an RFID status dashboard showing reader health, recent scans, and commissioning queue — content that actually belongs on a "landing" page.

Overall Assessment

This page adds friction without adding value. Users who click "RFID & Hardware" in the sidebar land here and then have to click again to reach actual tools. The page should either be removed (link the sidebar directly to EPC Tracker or Commissioning) or transformed into a real RFID health dashboard with useful at-a-glance information.
