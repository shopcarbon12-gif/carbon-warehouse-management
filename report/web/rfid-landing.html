RFID Workflows (Landing Page)

A simple navigation/landing page for RFID-related workflows. Provides three links: the external Carbon WMS legacy encode portal, the internal inventory search, and the handheld batch API documentation.

ℹ️ This is a landing page only — it has no functionality of its own, just links to other places. It could be replaced by better in-page navigation or removed if its content is accessible from the sidebar.

Links on This Page

Carbon WMS Legacy Encode (External Link)
External Link
Links to an external website (app.senitron.net) — the legacy RFID encoding portal. Opens in a new browser tab. Described as being used for "deep encode and LS PO flows."
Suggestion: Sending users to an external legacy system from within the main application creates a context-switch that can be confusing. The link should have a clear label that this is an external system ("External: opens in a new tab"), and ideally a note explaining when to use it vs. the internal commissioning tool.
Search Inventory Link
Internal Link
Links to the legacy Inventory page (/inventory).
Suggestion: This should point to the modern Inventory Catalog (/inventory/catalog) instead of the legacy inventory page.
Handheld Batch API Link
Internal Link
Links to the Handheld API documentation page (/handheld) which shows the curl command example.
Suggestion: This link is only useful for developers and device integrators — regular warehouse staff have no use for it. Consider moving this to a Developer Resources section separate from the main navigation.

Overall Landing Page Suggestions

1. This page adds little value: All three links are accessible from the sidebar. This landing page is an extra click with no real content. Consider removing it and making the RFID sidebar item go directly to the EPC Tracker (the most commonly used RFID tool).

2. If keeping the landing: Add tiles or cards for each RFID workflow with a brief description of what each one does — making it a useful "RFID quick start" hub rather than just a list of links.
