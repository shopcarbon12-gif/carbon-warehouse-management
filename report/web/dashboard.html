Dashboard — Command Center

The main overview page. Shows key numbers, hardware status, and a timeline of recent activity. Data refreshes automatically every 15 seconds. A live RFID scan counter updates in real time through a streaming connection to the hardware.

Page Header

"Command center / Operations overview" Header
Informational
Shows the title "Operations overview" with a subtitle explaining that KPIs refresh every 15 seconds. A "Syncing…" indicator appears briefly when data is being refreshed in the background. If the data fails to load, a red "KPI load error" message appears instead.
Suggestion: The subtitle contains technical language: "RFID edge stream (SSE) updates the live scan counter." For non-technical warehouse managers, this could be rewritten as "Live scan count updates automatically." Also, the "Syncing…" indicator is very small and easy to miss — a subtle progress bar across the top might be more noticeable.

KPI Cards (Key Numbers)

Total Active Inventory (Teal Card)
Clickable Card
Shows the total number of RFID-tracked items currently in the system for the active location. Clicking the card navigates to the Inventory Catalog page. While loading, the card shows an animated placeholder pulse. The number is a large bold count.
Suggestion: There is no trend indicator (e.g., up or down arrow compared to yesterday). A simple "+12 since yesterday" indicator would make this number much more useful. Also, the card title "Total active inventory" is clear, but adding a brief description like "items with in-stock RFID tags" would help new users understand what they are looking at.
Receiving Concerns (Amber Card)
Clickable Card
Shows the count of items that have been flagged as having receiving issues — items that arrived but have discrepancies. Clicking the card goes to the Alerts page where these can be reviewed and resolved.
Suggestion: The label "Receiving concerns" is somewhat vague. A number of "5" here raises the question: 5 what exactly? Labeling it "5 items flagged during receiving" or adding a tooltip explaining what a "receiving concern" is would be clearer for users new to the system.
Unknown Assets (Violet Card)
Clickable Card
Shows the count of RFID tags that have been scanned by readers but do not match any known item in the database. These could be tags from another system, damaged tags, or items that were never properly registered. Clicking goes to the Inventory page.
Suggestion: Unknown assets can be a sign of a serious problem (items slipping through without being cataloged) or a minor one (a stray tag from a box). The card would be more actionable if clicking it went directly to a filtered view showing only the unknown assets, not the entire inventory.

Hardware Pulse Strip

Live Scans (Session) Counter
Live Counter
Shows how many RFID tags have been scanned by the fixed readers since you opened this page. It updates in real time via a streaming connection (Server-Sent Events). If readers are active and scanning, this number ticks up continuously. The pill glows green when the count is greater than 0.
Suggestion: The counter resets to 0 every time you reload the page (it is session-based). This means it is not useful for tracking how many scans happened during a workday. Consider showing both "this session" and a daily total. Also, the counter never resets itself — if 10,000 items are scanned over a long shift, it just shows a large number with no context.
Hardware Pill Cards (Readers, Antennas, Printers, Handhelds)
Status Display
Four fixed-count pills showing the number of each hardware type. Currently hardcoded to: 1 Reader, 9 Antennas, 1 Printer, 0 Handhelds. Green glowing dots appear next to types with a count greater than 0. The numbers are static — they do not reflect real-time device status.
Suggestion: These numbers are hardcoded in the source code, not pulled from the device registry. This means they will never change even if you add or remove hardware. This section should connect to the live device registry to show actual device counts. Also, "0 Handhelds" with no context looks like a problem — it should either show something like "Not connected" or be hidden when zero.

Recent Activity Timeline

Activity List (Last 10 Audit Events)
Timeline
Shows the 10 most recent things that happened in the system — tag scans, status changes, imports, logins, etc. Each item shows what happened, which entity was affected, and the exact date/time. Items are connected by a vertical line to create a timeline effect. If nothing has happened yet, it shows "No audit events yet."
Suggestion: The entries are formatted as machine-readable text (e.g., "update · rfid_asset — detail here"). For a warehouse manager, this should be rewritten in plain language: "Tag E280... moved from Bin A to Bin B" or "Status changed for SKU-123 from In Stock to Sold." Also, there is no way to filter or search the activity from here — you have to go to the full Activity History report.
"View all" Link
Link
A small link at the top of the activity panel that takes you to the full Activity History report page where you can see up to 200 audit events.
Suggestion: This is good. Consider also allowing inline filtering — for example, a dropdown on the dashboard activity to show only "Today's activity" or filter by event type.
Quick Links Bar (Bottom of Activity Section)
Link Bar
A small row of links at the bottom of the activity section: "Inventory · Compare · Integrations · Commissioning". These are shortcuts to commonly used pages.
Suggestion: These links are barely visible — they sit in a grey box below the timeline and look like secondary content. They could be styled as larger, more prominent action buttons to serve as a proper "quick actions" area for the most common daily workflows.

Overall Dashboard Suggestions

1. Refresh behavior: The dashboard auto-refreshes every 15 seconds, but there is no visible "last updated at [time]" indicator. Adding this gives users confidence the data is current.

2. Date/time context: The dashboard shows no current date or time. In a warehouse that runs multiple shifts, this matters — staff should see "Today: Monday April 11" prominently.

3. Make KPI cards actionable: Each KPI should link to a page that shows the relevant data, ideally pre-filtered. Currently, "Unknown Assets" links to the general Inventory page — it should link to a filtered view of unknowns.

4. No "nothing to do" state: When everything is fine (0 concerns, 0 unknowns), the dashboard should congratulate or reassure the user: "All good — no open exceptions." Right now it just shows 0s without any context.

5. Hardware hardcoding: Fix the hardware pill counts to pull from the real device registry rather than being hardcoded.
