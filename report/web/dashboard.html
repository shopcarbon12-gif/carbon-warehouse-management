KPI Tiles

KPI Tiles — Catalog · Alerts · RFID In-Stock
Clickable Cards
Three large tiles at the top. Catalog Items count links to Inventory Catalog, Open Alerts count links to the Alerts page, RFID In-Stock count links to Inventory. All three auto-refresh every 15 seconds.
Clicking a tile to navigate directly to the relevant page is a natural, efficient design. Auto-refresh keeps numbers accurate without user action.
The Alerts tile looks the same whether there are 0 or 50 open alerts — it does not change color or show any urgency. A manager glancing at the dashboard cannot tell at a glance if something is wrong.
Make the Alerts tile turn amber/red when the count is above zero. Consider adding a small change indicator to the stock tile (↑ +12 today) so the number has context. Numbers without context are just numbers.

Hardware Pulse Strip

Hardware Status Row (Readers · Printers · Handhelds)
Status Strip
A row of indicator dots showing connected hardware — RFID readers, printers, and handhelds — with counts and online/offline labels.
The counts shown here are hardcoded in the source code — they are not pulled from the real device registry. A reader going offline does not change this display. This is the most misleading element in the entire web app: operators making floor decisions based on "3 readers online" when that number is fake.
Pull device counts and status from the Infrastructure Devices registry API. Show actual device names with live Online/Offline states. Remove the hardcoded values entirely — displaying wrong data is worse than displaying no data.

Live Scan Counter

Live Scan Stream Counter
Real-Time Feed (SSE)
A counter that increases in real time as RFID tags are scanned anywhere in the warehouse. Uses a persistent server connection (SSE) that pushes events as they happen — no polling needed.
Real-time push without polling is the right technical approach. Operators can immediately see that scanning activity is happening. The live nature makes the dashboard feel alive and trustworthy.
Only a total count is shown. A number like "1,247 scans" with no time frame, location breakdown, or comparison to yesterday is hard to interpret. The user can't tell if this is normal or abnormal activity.
Add a compact activity graph (sparkline) for the last 30 minutes so a sudden drop or spike is immediately visible. Show "today" and "last hour" sub-counts. This turns a raw number into a useful signal.

Activity Timeline

Activity Timeline — Last 10 Events
Audit Feed
A list of the 10 most recent system events — status changes, sync completions, user actions — with timestamps. Refreshes every 15 seconds.
Surfacing recent activity on the home screen lets a manager catch unexpected actions without opening full audit logs.
10 events disappear quickly during busy periods. There is no user name shown per event — a manager can see "Status changed to damaged" but not who did it. The audit feed is also not filtered, so routine high-volume scan events crowd out meaningful actions.
Increase to 20 entries. Add who performed each action ("by john@..."). Filter out repetitive bulk scan events and show only meaningful actions: status changes, user logins, sync results, device authorization. Add a "View full log →" link at the bottom.

Quick Links Bar

Quick Links
Navigation Shortcuts
A row of shortcut links to commonly used pages — Inventory, Commissioning, Cycle Counts, etc. Same links for every user regardless of role.
A receiving-team member sees Commissioning shortcuts. A manager sees Cycle Count shortcuts. Irrelevant links add clutter and slow down navigation for everyone.
Show quick links based on the logged-in user's role. A floor supervisor → Exceptions and Cycle Counts. A receiving team member → Commissioning and Barcode Intake. Role-specific shortcuts reduce noise and help new staff find their workflow immediately.

Overall Assessment

Critical: hardcoded hardware strip. This must be fixed before anything else — displaying fabricated device counts to operators who make real decisions based on them is an active operational risk.
Priority fix order:
1. Replace hardcoded hardware strip with live device data.
2. Add color/urgency to the Alerts tile when count > 0.
3. Add "by [user]" to activity timeline entries.
4. Role-based quick links.
