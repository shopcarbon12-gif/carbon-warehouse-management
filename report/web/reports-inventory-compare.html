POS Compare Report

The most important reconciliation report. Compares what Lightspeed POS says should be in stock (expected quantity) versus what RFID tracking shows is actually in stock (found quantity). Highlights which items are missing or have extras, with CSV export and integration actions.

Summary Tiles

Summary Stats (Total Expected, WMS Total, Missing, Extra)
KPI Display
Shows four aggregate numbers at the top: total items expected by POS, total found by RFID, total missing (in POS but not in RFID), and total extras (in RFID but not in POS). These give an instant health check of inventory accuracy.
Suggestion: Add a simple accuracy percentage: "Inventory accuracy: 94.7%" (found ÷ expected × 100). This is the metric warehouse managers actually care about. Also show a trend: "Up 2.3% since last compare."

Comparison Table

SKU Comparison Rows (variances only)
Data Table
The table by default shows only rows where there is a variance (missing > 0 OR extra > 0). Each row shows: SKU, item name, LS Item ID, expected quantity from Lightspeed, found quantity in RFID, missing count, and extra count. Rows can be selected with a checkbox.
Suggestion: Show ALL rows by default (not just variances) with a toggle to "Show variances only." Users may want to confirm that correct items have zero variance. Also, color-code the missing/extra columns: red for missing, yellow for extra.
Row Checkboxes (Select for Action)
Checkbox
Each row has a checkbox. Selecting rows enables bulk actions (like sending selected variances to an integration or marking them as reviewed).
Suggestion: Add a "Select all variances" button to quickly select all problematic rows without having to check each one individually.

Action Buttons

Pull from Lightspeed Button
Button
Triggers a fresh pull of POS quantity data from Lightspeed to update the comparison data. Shows the result of the pull (source and whether data was real or simulated).
Suggestion: This button should show the timestamp of the current POS data being used ("POS data last updated: 2 hours ago") so users know if they need to pull fresh data before comparing.
Export CSV Button
Button
Downloads all comparison rows as a CSV file with columns: SKU, Name, LS_ITEM_ID, Expected_LS, Found_WMS, Missing, Extra. The filename includes today's date.
Suggestion: Allow exporting only selected rows, or only variance rows, with a toggle. Also add a "Download full comparison (all items)" option separate from "Download variances only."
Send to Integration Button (for selected rows)
Button
For selected rows, triggers an integration action (likely pushing adjustment data back to Lightspeed or an accounting system). Shows result message.
Suggestion: This is a high-stakes action that pushes data to an external system. It must show a clear confirmation dialog: "You are about to send 12 variance adjustments to Lightspeed. This will modify stock quantities in your POS. Continue?" And it should require at least admin-level access.

Metadata / Source Info

Meta Section (hints, endpoints)
Info Panel
Shows metadata about the comparison: where the expected qty came from, any hints or warnings, and API endpoint names. This is technical information.
Suggestion: Replace technical endpoint names with plain language: "POS data source: Lightspeed R-Series (synced 2 hours ago)." Remove raw API endpoint paths from the user-facing UI.
