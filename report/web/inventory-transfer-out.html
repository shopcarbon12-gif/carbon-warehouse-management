Transfer Out

Creates outbound transfer slips — documents that record which items are being sent from one location to another. Staff create a slip, optionally add specific RFID EPCs, then export or print it as a packing list for the floor.

Create New Slip Form

Source Location Code Field
Text Input
A text field where you type the location code for where items are being sent FROM (e.g., "STORE-A"). This is a free-text field — no dropdown of existing locations. Required to create a slip.
Suggestion: A free-text field for location codes is error-prone — typos will create slips with invalid locations that can't be matched later. This should be a dropdown/searchable list populated from actual warehouse locations in the system. The field label says "Source location code" which is technical — "Sending From" would be clearer.
Destination Location Code Field
Text Input
Same as above but for the location where items are going TO. Also free-text, also required.
Suggestion: Same improvement needed as Source field — replace with a dropdown list of real locations. Consider renaming to "Sending To."
EPCs Textarea (Optional)
Textarea
Optional field where you paste specific RFID EPC codes of the items being transferred. Accepts whitespace, commas, semicolons as separators. If left empty, the slip is created without pre-assigned items (they can be scanned on the floor later).
Suggestion: The label says "EPCs (optional, whitespace-separated)" — consider also supporting scanning from a handheld to populate this field. Also, a live count of detected EPCs would be helpful.
Generate Transfer Slip Button
Primary Button
Creates the slip in the database. Slip numbers start from 70,000+. After creation, a success message shows the slip number (e.g., "Created slip #70001"). The slips list below refreshes automatically.
Suggestion: After creating a slip, the user likely wants to immediately print or export it. Add a shortcut button after creation: "View & Print Slip #70001" instead of making them find it in the list below.

Slips List Table

Slips Table
Data Table
Shows all existing transfer slips sorted newest first. Columns: Slip number, Source location, Destination location, Status, LS Transfer ID (Lightspeed reference), and Actions. Each row shows the current status of the slip (e.g., open, completed).
Suggestion: The table has no filtering — if there are hundreds of slips, finding a specific one requires scrolling. Add search by slip number or location, and filter by status (Open/Completed/All). Also, the "LS" column header is cryptic — spell it out as "Lightspeed ID."
Select Button (per row)
Button
Clicking "Select" on a slip highlights that row. It is not entirely clear from the code what selecting a slip enables beyond the visual highlight — it may be a precursor to additional actions.
Suggestion: If "Select" is a stepping-stone to an action, the purpose should be clearer. Consider showing what actions become available after selecting a slip, or replacing the select mechanism with inline action buttons per row.
CSV Button (per row)
Button
Downloads the packing list for that slip as a CSV file. The CSV contains: ALU (item code), Name, Sent quantity, Received quantity, Missing quantity, and EPC codes. The file is named "transfer-slip-70001.csv".
Suggestion: This is the most useful action. Consider making it the most prominent button (larger, with a download icon). The CSV column headers could be more descriptive — "ALU" means nothing to most warehouse workers; "Product Code" would be clearer.
PDF/Print Button (per row)
Button
Opens a new browser window with a simple HTML table of the slip's contents, then automatically triggers the browser's print dialog. The printout is a basic table with slip number, item details, and quantities. Designed to be printed and handed to warehouse staff.
Suggestion: The printout is very plain HTML. Consider adding a logo, the warehouse name, date/time of printing, and space for a signature at the bottom. Also, the print window opens in a new tab, which may be blocked by popup blockers — add a fallback message.
