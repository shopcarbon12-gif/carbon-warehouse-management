Transfer Out

Creates outbound transfer slips — numbered from 70000+ — listing items being sent from this location to another. Has a form to create new slips (with free-text destination fields and optional EPC list), a table of all existing slips, and per-row CSV export and print-to-PDF buttons.

Create Transfer Slip Form

Destination Location Fields
Free-Text Inputs
Two free-text fields for the destination: location name and location code. No validation against known locations. A user can type anything.
Free-text destination fields mean typos are possible and there is no link between a transfer slip and an actual destination location in the system. A slip going to "Maim Warehouse" (typo) creates a record that can never be matched to a real location.
Replace with a dropdown of known locations from the Settings → Locations list. The location code should auto-fill when a name is selected. For locations outside the system (external vendors, other companies), allow a free-text "external destination" option as a fallback.
EPC List (Optional)
Textarea
An optional textarea for pasting EPC codes to include in the slip. If left empty, the slip is created without items, and items can be added later via scanning.
Allowing slip creation without a pre-defined item list gives flexibility for workflows where items are added during packing rather than before. Both workflows (pre-defined list and scan-as-you-go) are supported.
If EPCs are pasted, there is no validation or preview — the user can't confirm which items they are before creating the slip.
After EPCs are entered, show a count and a sample of item names: "12 items entered — e.g. Red Jacket XL, Blue Jeans 32W…" Let the user confirm before creating.

Slips Table & Actions

Transfer Slips Table
Data Table
Lists all transfer slips created from this location. Columns: slip number (70000+), destination, item count, created date, status. Each row has buttons for CSV export and Print PDF.
Slip numbers starting at 70000 are easy to communicate verbally — "send me transfer 70043" is much more practical than a UUID. The CSV and PDF actions per row are conveniently placed.
No ability to search or filter by destination or date range. With many slips, finding a specific one requires scrolling through the entire list.
Add a search/filter bar above the table. Filter by destination location, date range, and status (open/received). This makes the Transfer Out page useful as an historical reference, not just a creation tool.

Overall Assessment

Free-text destination fields break the link between transfer records and the actual location system. Replace with a dropdown of known locations immediately — this is a data quality issue that will cause problems when trying to track transfers historically.
