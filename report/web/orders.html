Page Header

Page Title, Subtitle & Quick Links
Header
Shows "Orders & Fulfillment" with the warehouse name. A description says "pull from Lightspeed and internal sources; pick by zone, pack, ship." Three quick links: Integrations, Inventory Sync, Dashboard.
The three quick links (Integrations, Inventory Sync, Dashboard) provide useful navigation shortcuts from a page that will often be referenced alongside those tools.
The page description promises pick/pack/ship workflows that do not exist on this page. Setting that expectation in the subtitle and then showing a bare read-only table creates confusion about what the page is for right now.
Either replace the description to reflect the current capability ("View incoming orders from Lightspeed"), or add a clear "Coming soon" note for the pick/pack/ship workflow so users know it's planned, not missing.

Orders Table

Orders Table
Data Table
Five columns: Ref, Source, Status, Lines (count), and Created date. No sorting, no filtering, no pagination. All orders load at once. Clicking a row does nothing.
With no filtering, sorting, or pagination, this table becomes unusable as order volume grows. A warehouse with hundreds of orders sees them all in one unsorted list with no way to focus on what matters.
Clicking an order row does nothing — there is no order detail view, no way to see line items, no way to take any action on an order.
Add at minimum: status filter (Open / In Progress / Completed / All), sort by date, and a row click that shows order details with line items. These are the baseline features expected of any order management page.
Status Display
Table Column
Status values displayed as raw database strings with underscores replaced by spaces (e.g., "pending pick" from "pending_pick"). No color coding.
Status labels without color coding require reading every cell to understand which orders need attention. "Open" and "Pending Pick" and "Completed" all look the same visually.
Color-code status badges: "Open" = green, "Pending Pick" = amber, "Completed" = grey. This allows an operator to immediately identify which orders require action without reading each row.
Empty State
Empty State
Shows: "No orders — connect database or sync from Lightspeed." Links to the two required actions.
"Connect database" is developer language that has no meaning to a warehouse admin. The system is already connected to a database — this message will confuse operators who don't know what they're supposed to do.
Replace with: "No orders found. Sync from Lightspeed to pull the latest orders." Format the sync link as a button rather than plain text so the call to action is obvious.

Overall Assessment

The Orders page is in an early-stage state — it displays order data but provides no way to interact with or act on any order. The promised pick/pack/ship workflow does not yet exist.
Key improvements:
1. Add filter by status and sort by date — baseline for any order list.
2. Make rows clickable to show order detail with line items.
3. Color-code status badges (Open/In Progress/Completed).
4. Either remove or update the description that promises unfulfilled pick/pack/ship workflows.
