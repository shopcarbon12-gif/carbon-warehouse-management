Orders & Fulfillment

Displays a list of orders pulled from Lightspeed or internal sources. Currently a read-only table showing order references, source, status, number of lines, and creation date. The fulfillment workflow (pick, pack, ship) appears to be planned but not yet implemented.

ℹ️ This page appears to be in an early stage. It shows order data but has no interactive fulfillment actions yet — no picking, packing, or shipping workflows are present.

Page Header

Page Title & Subtitle
Header
Shows "Orders & Fulfillment" with the warehouse name below it. A description says "pull from Lightspeed and internal sources; pick by zone, pack, ship." Three quick links: Integrations, Inventory Sync, Dashboard.
Suggestion: The description promises pick/pack/ship workflows that do not yet exist on this page. Either implement them or remove the description to avoid setting false expectations. The quick links are useful but could be styled more prominently as action buttons.

Orders Table

Orders Table
Data Table
Five columns: Ref (external order reference or internal ID), Source (where the order came from — e.g., "lightspeed"), Status (e.g., "open", "completed", "pending_pick"), Lines (count of line items), and Created date. No sorting, no filtering, no pagination. All orders load at once.
Suggestion: With no filtering, this table becomes unusable once order volume grows. At minimum add status filtering (Show: Open / In Progress / Completed / All) and sort by date. Also, clicking a row should open order details — currently rows do nothing when clicked.
Status Display
Table Column
Status values are displayed as raw database strings with underscores replaced by spaces (e.g., "pending pick" from "pending_pick"). No color coding.
Suggestion: Status labels need color-coding: "Open" = green, "Pending Pick" = yellow, "Completed" = grey/blue. A status without visual differentiation is hard to scan quickly.
Empty State
Empty State
Shows: "No orders — connect database or sync from Lightspeed." Links to the two required actions.
Suggestion: The empty state message is acceptable. Add direct links: "Go to Integrations" and "Go to Inventory Sync" formatted as buttons rather than plain text.

Overall Orders Page Suggestions

1. Fulfillment workflow: The page description promises pick/pack/ship functionality. Until this is built, the page is just a read-only list. Consider adding a note: "Picking workflow — coming soon."

2. Order detail view: Clicking an order should open a detail view showing all line items, item quantities, and picking status per item.

3. Manual order creation: Allow creating internal orders (not just syncing from Lightspeed) for warehouse transfers or internal fulfillment needs.

4. Filter and sort: Essential for any order management page. Add at minimum: status filter, date range filter, and sort by date/status.
