Bulk Imports Report

An audit trail for bulk CSV and EPC import operations. Every time a large batch of items is imported (via file upload or batch API), a record is created here showing how many rows were processed and which succeeded or failed.

Bulk Imports Table

Import Job Log
Data Table
Shows import job records. Typical columns: Job ID or timestamp, import type (CSV, API), total rows submitted, rows successfully processed, rows failed, and status. Uses the bulk imports workspace component.
Suggestion: For failed rows, show exactly what went wrong — a "View errors" drill-down per job would let operators fix and re-import the failed items. Currently there is no way to know which specific items failed without checking the raw database.

Overall Bulk Imports Suggestions

1. Re-import failed rows: Add a "Retry failed rows" button per job that re-processes only the items that failed the first time.

2. Download original file: Allow downloading the original file that was uploaded so operators can reference what was submitted.

3. Notify on completion: For large import jobs that take time to process, an email or in-app notification when the job is done would be more user-friendly than having to check this page manually.
