Status & Tag Logs Report

A history of all status changes applied to RFID tags. Every time an item's status changes (e.g., from "In Stock" to "Sold," or from "Return" to "In Stock"), an entry is recorded here with the old status, new status, who made the change, and when.

Data Table
Loads status change history. Shows EPC, item details, old status, new status, changed by (user or system), and timestamp. Uses the shared status-logs workspace component.
Suggestion: Add filtering by: status transition (e.g., "show all items that went from In Stock to Damaged"), date range, user who made the change, and specific EPC or SKU. This report is most powerful as an investigation tool — filtering is essential.

Overall Status Logs Suggestions

1. Suspicious pattern detection: Flag unusual status patterns — e.g., an item going from "In Stock" to "Sold" and back to "In Stock" multiple times in one day. This could indicate a scan error or data problem.

2. Bulk change visibility: When a bulk status update changes 200 EPCs at once, those 200 entries all appear in this log. They should be grouped visually as one bulk operation with an expand option, rather than 200 individual lines.
