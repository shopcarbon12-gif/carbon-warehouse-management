Bulk Status Update

Lets authorized users paste a list of RFID EPC codes and change all of their statuses at once. Regular users can only use safe status options. Super Admins have access to all statuses including dangerous system ones.

Role Note

Role Warning Text
Informational
A small note at the top explains what the user can and cannot do based on their role. Regular users see: "System workflow targets are hidden — Super Admin only." Super Admins see: "All Clean 10 targets are available; use Override for risky transitions."
Suggestion: The term "Clean 10" is an internal system vocabulary. For most staff users, this means nothing. Consider using plain language or providing a tooltip/link explaining what these statuses mean.

Form Controls

Target Status Dropdown
Dropdown
A dropdown to pick the new status you want to apply to all the pasted EPCs. Regular users see only safe, staff-facing statuses (like "In Stock", "Sold", etc.). Super Admins see all statuses including system-internal ones. The first option in the list is selected by default.
Suggestion: There is no preview of what the selected status means or what happens when it is applied. A brief description under the dropdown (e.g., "In Stock: item is available for sale and visible to RFID readers") would help users choose correctly. Also, consider grouping statuses visually — e.g., "Normal statuses" vs "Super Admin statuses."
Allow Risky Transitions Checkbox
Checkbox
A checkbox labeled "Allow risky transitions (Super Admin only)". It is disabled (greyed out, cannot be checked) for regular users. For Super Admins, checking it allows moving items between statuses that would normally be blocked — for example, moving something from a locked system state back to a normal state.
Suggestion: When checked, there is no second confirmation or warning before running the bulk operation. Given that this can affect many items at once and the changes are described as "risky," a confirmation dialog summarizing what will happen is essential: "You are about to move 450 EPCs from [status] to [status]. This cannot be automatically reversed. Continue?"
EPC Text Area
Textarea
A large text box where you paste or type EPC codes. EPCs can be separated by spaces, commas, semicolons, or new lines — the system handles all of these automatically. The box accepts any formatting and cleans it up before processing. Minimum height of 180px.
Suggestion: There is no live count showing how many EPCs have been detected in the pasted text. Adding "Detected: 47 EPCs" that updates as you type would give users confidence they pasted the right amount. Also, consider adding a "Clear" button inside the textarea and a "Paste from clipboard" button for convenience.
Apply Bulk Status Button
Primary Button
A teal button that submits the operation. It shows "Working…" while processing and is disabled during that time. After completion, a message appears showing how many EPCs were updated (e.g., "Updated 47 EPC(s) at the active location.").
Suggestion: After a successful bulk update, there is no way to undo the operation. For regular users this is fine since they can only pick safe statuses. But for Super Admins using the Override option, a brief undo window (e.g., 30 seconds with an "Undo" button) could prevent costly mistakes. Also, the success message disappears if you click elsewhere — it should persist until dismissed.

Overall Bulk Status Suggestions

1. Preview before applying: Show a summary of how many EPCs were detected and what the current vs target status would be before running the operation.

2. Result breakdown: After running, show which EPCs succeeded, which were skipped (already in target status), and which failed (not found or locked).

3. Activity log link: After a bulk update, show a direct link to the relevant audit log entries so the user can verify what was changed.

4. File upload option: Allow uploading a CSV or TXT file of EPCs instead of only pasting — useful when dealing with hundreds of codes from a scanner export.
