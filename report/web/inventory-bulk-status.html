Bulk Status Update

A power-tool page for admins. Paste a list of EPC codes, choose a target status, and apply the change to all of them at once. Some dangerous status transitions are behind an additional Super Admin checkbox. This page can change the status of thousands of items in one action.

Input Form

Target Status Dropdown
Dropdown
Shows all 10 Clean 10 statuses. Selecting one sets the target status for the bulk operation. Some statuses only appear if the "risky transitions" checkbox is checked and the user is a Super Admin.
Hiding dangerous statuses behind a separate Super Admin checkbox is the right two-layer protection for destructive operations.
Status names use internal identifiers (in-stock, tag_killed, pending_visibility) that are not self-explanatory to all admin users. A new admin may not know the difference between "sold" and "tag_killed."
Show a one-line description next to each status in the dropdown: "tag_killed — RFID chip deactivated at checkout; scanner will ignore it." This prevents mis-selection.
EPC Textarea
Large Text Area
A large text area where the admin pastes EPC codes, one per line. There is no limit shown on how many EPCs can be entered. No validation of EPC format is shown before submission.
No count is shown as the user types — pasting 500 EPCs looks the same as pasting 5000. There is no format validation, so a pasted list with trailing spaces, empty lines, or wrong format silently fails per EPC. No preview of what items will be affected before committing.
Show a live count: "437 EPCs entered" as the user types/pastes. Validate format and show how many are recognized: "435 valid · 2 not found in this tenant." Add a preview step showing the items before applying the change.
Risky Transitions Checkbox (Super Admin only)
Checkbox (Role-Gated)
Only visible to Super Admins. Checking it unlocks additional status options in the dropdown that are considered dangerous — like statuses with Super Admin locks. The checkbox is hidden entirely for regular admins.
Role-based visibility of dangerous options is exactly the right pattern. Regular admins can't even see the risky transitions option exists.
Apply Button
Primary Button (Destructive)
Immediately applies the selected status to all EPCs in the textarea. No intermediate confirmation step. After completion, shows success or error counts.
There is no confirmation dialog before applying. A mis-paste or a wrong status selection applies immediately. Changing the status of 1,000 items and then discovering the wrong status was selected has no undo path.
Add a required confirmation step: show the target status, the count of EPCs, and a preview of 3–5 item names before applying. The confirm button should read "Apply 'damaged' to 437 items" — making the user read what they are about to do. Add an audit log entry for every bulk operation with the before/after states.

Overall Assessment

No confirmation before a bulk destructive operation is the critical problem here. This page can change thousands of inventory records in one click. A single bad paste or wrong dropdown selection with no review step is an unacceptable risk for production inventory data.
Must-fix before next release:
1. Add a preview/confirmation step before applying.
2. Show live EPC count as user types.
3. Validate EPC format and show found/not-found counts.
4. Log all bulk operations with before/after state in the audit trail.
