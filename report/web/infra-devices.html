Infrastructure Devices

The central registry of all physical hardware connected to CarbonWMS — RFID readers (fixed and handheld), label printers, and other devices. Each device is assigned to a warehouse location and optionally to a specific bin. This page is the source of truth for what hardware exists and where it is.

Filter Tabs

Device Type Filter Tabs
Tab Bar
A row of tab buttons that filter the devices table by type: "All" (shows everything), plus one tab per device type (e.g., Handheld Reader, RFID Reader, Printer). The active tab is highlighted with the accent color. Clicking a tab immediately filters the list without reloading.
Suggestion: The device type tabs do not show a count badge (e.g., "Handheld Reader (3)"). Adding counts would let admins quickly see how many of each type are registered without switching tabs. Also, the tab labels come from internal type names like "handheld_reader" with underscores replaced by spaces — ensure the display names are user-friendly (e.g., "Handheld Scanners" not "handheld reader").

Devices Table

Devices Table
Data Table
Shows all registered devices in a table. Columns: Name (the device's friendly name), Type (the device category), IP / MAC (the network address — truncated with full text on hover), Location (location code and name), Bin (the specific bin assigned — "—" if none), Status (Online in green or Offline in grey strikethrough), and action buttons (Edit + a trash icon for Remove).
Suggestion: The "Online/Offline" status is shown but there is no indication of when this status was last checked or how it is determined. Is it a live ping, or just what was last reported? Add a "Last seen" timestamp next to the Online/Offline indicator (e.g., "Online · 2 min ago" or "Offline · last seen 3 days ago"). This helps distinguish between a device that is intentionally powered off vs one that has gone missing.
Empty State
State
When no devices match the selected filter tab, a centered empty state appears with a chip icon, the message "No devices registered in this category," a brief explanation ("Add your first Zebra printer or Carbon edge reader. Locations come from the warehouse map — assign each device so scan events can be attributed to a site (and optionally a bin)."), and a "Register a device" button.
Suggestion: The empty state message mentions "Zebra printer" and "Carbon edge reader" by name — this is good contextual guidance. Keep this language but ensure it matches the actual hardware used in the deployment. The empty state is well designed and actionable.
Add Device Button
Primary Button
Opens a device editor modal for creating a new device entry. The modal allows setting the device name, type, network address, location, and bin. On save, the device is added to the registry and appears in the table.
Suggestion: The Add Device button and the "Register a device" empty state button both open the same modal — which is consistent. Consider adding a brief help note in the modal explaining what information is needed and why: "The network address (IP or MAC) is used to send print jobs to printers and to identify which reader reported a scan." Without context, admins may not know where to find these values.
Edit Button (per device)
Button
Opens the device editor modal pre-filled with the device's current settings. All fields are editable.
Suggestion: When editing a device that is currently Online, warn the admin that changes (like updating the network address or reassigning to a different location) may affect live scanning: "This device is currently active. Changes will take effect immediately and may briefly interrupt its connection."
Remove Button (per device)
Danger Button (trash icon)
Shows a browser confirmation dialog ("Remove '[name]' from the registry?") before deleting. On confirmation, the device is removed. No undo available.
Suggestion: Removing a device deletes it from the registry, which may affect scan attribution for any scans that come in from that device afterwards. Use a more descriptive confirmation: "Remove [Name]? Future scans from this device will not be attributed to a location. This cannot be undone." Also, if the device is currently Online, block or at least strongly warn: "This device appears to be online. Are you sure you want to remove it?"

Overall Infrastructure Devices Suggestions

1. No real-time health monitoring: The page shows Online/Offline status but no connection history, error logs, or metrics. For a production warehouse, knowing "this reader has been dropping off every evening at 8 PM" is critical for maintenance. A simple connection history log per device would help.

2. No printer test: For registered printers, a "Send test print" button would allow admins to quickly verify the printer is connected and working without going to the commissioning page.

3. Device grouping by location: In a multi-location deployment, the flat table becomes hard to navigate. Allow grouping devices by location so admins can see all devices at "Main Warehouse" together.
