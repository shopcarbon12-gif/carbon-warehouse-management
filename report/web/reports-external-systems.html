External Systems Report

A log of all communication events between CarbonWMS and external systems — outgoing API calls, webhook deliveries, sync events, and responses received. Useful for diagnosing integration issues.

External System Events Table

Event Log
Data Table
Shows events for all integrations: system name, event type, direction (outbound/inbound), status (success/failure), and timestamp. Uses the external systems report workspace component.
Suggestion: For failed events, show the error details (HTTP status code, error message) to allow diagnosis without needing to check server logs. Also add filtering by system (Lightspeed, etc.), event type, and status (success/failure only).

Overall External Systems Suggestions

1. Alerting on failure: Consecutive failures from an external system should trigger an alert visible on the Dashboard and in the Alerts section. Currently failures are only discoverable by checking this report manually.

2. Retry button: For failed outbound events, a "Retry" button would allow re-sending without having to find and re-trigger the original action.
