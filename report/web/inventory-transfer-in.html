Transfer In

For receiving incoming shipments against existing transfer slips. Shows the same slips list as Transfer Out but without the ability to create new slips. The purpose here is to track what was received vs what was expected.

ℹ️ This page shares the same component as Transfer Out, but in "in" mode — so it shows the same slips list but hides the "Create new slip" form at the top.

Slips List Table

Same Slips Table as Transfer Out
Data Table
Shows all transfer slips sorted newest first. Same columns as Transfer Out: Slip #, Source, Destination, Status, LS ID, and Actions. The description says "Open an existing slip and export receiving progress. Reconcile counts on the handheld, then update statuses in admin when complete."
Suggestion: Since this page is specifically for receiving, consider filtering the slips to show only ones that are "in transit" (i.e., waiting to be received at this location). Showing all slips, including already-completed ones, clutters the view for receiving staff. Also, the page title says "Transfer in" but there is no visual difference from "Transfer out" — a different highlight color or a clear label on each row showing direction (IN or OUT) would help.
CSV Export Button
Button
Downloads receiving progress — shows what was sent vs what was received vs what is still missing. This is the main tool for reconciling a received shipment.
Suggestion: The CSV export is useful for reconciliation, but it is a passive tool — the user has to manually compare. Consider adding an inline "mark as received" feature that lets receiving staff check off items directly in the web UI rather than going back and forth with a spreadsheet.
Print Button
Button
Same print functionality as Transfer Out — opens a new browser tab with an HTML table and auto-triggers the print dialog.
Suggestion: A "receiving copy" of the slip should visually differ from the "sending copy" — different header, different columns visible. The receiving slip should emphasize the "Received" and "Missing" columns rather than "Sent."

Overall Transfer In/Out Suggestions

1. Separate views for in vs out: Currently both pages show identical tables. The Transfer In page should show a different view optimized for receiving — e.g., sorted by "incoming to my location" first, with a "Mark Received" status button.

2. Status updates: There is no way on the web to update a slip's status from "in transit" to "received." The description mentions doing this "in admin when complete" — but there is no admin status update button visible on this page.

3. Barcode scanning to match: Receiving staff should be able to scan items with a barcode scanner and have the system automatically match them against the expected EPC list.
