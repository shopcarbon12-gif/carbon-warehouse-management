Inventory — Legacy View

An older flat inventory table, kept alongside the newer Catalog page. Has a filter form with text search and zone filter, shows up to 80 rows, and is read-only. No actions can be taken from this view.

Filter Form & Table

Filter Form (q= and zone=)
Search Form
A search form with a text query field and a zone dropdown. Submitting the form reloads the page with filter parameters in the URL. Results are limited to 80 rows with no pagination.
An 80-row hard limit with no pagination means any search returning more than 80 items silently drops the rest. Users have no way to know results were cut off, and no way to see the remaining items. This is a data integrity problem — staff may conclude an item doesn't exist when it just wasn't in the first 80 results.
Either add pagination or, better, migrate users entirely to the Inventory Catalog page which handles this properly. The legacy page should show a clear notice: "Showing first 80 results — use the Inventory Catalog for full search."
Empty State Message
State
When no items match the filter, a message is shown. The message text is developer-authored and intended for internal use.
The empty state text is written for developers, not warehouse staff. It contains technical references that would confuse a floor worker who just searched for an item and found nothing.
Replace with: "No items found matching your search. Try different keywords or check the Inventory Catalog for a full search."
Read-Only Table
Data Table
The table is display-only. No actions, no exports, no sorting, no clicking into an item detail. It exists purely to view data.
A completely read-only page with a hard 80-row limit and no export means it is nearly useless compared to the Inventory Catalog page, which has all these features. There is no reason for both to exist simultaneously.
Deprecate and eventually remove this page. Add a prominent banner at the top: "This is an older view. For full inventory search, export, and actions, go to the Inventory Catalog." Link directly to the Catalog.

Overall Assessment

This page should be removed or replaced with a redirect to the Inventory Catalog. It offers strictly less functionality than the Catalog page, has a hard data limit that silently drops results, and the empty-state text is developer language. Keeping it in the sidebar creates confusion about which inventory view to use.
Recommended action: Add a top-of-page banner directing users to the Inventory Catalog. Plan removal within the next development cycle. There is no UX benefit to maintaining two separate inventory views with different capabilities.
