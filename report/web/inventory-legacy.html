Inventory (Legacy View)

An older, simpler inventory table. Filterable by keyword and zone. Shows up to 80 items. Read-only. Being replaced by the Inventory Catalog.

⚠️ This page appears to be the older version of the Inventory Catalog. It is accessible via a direct link from the Catalog page. Consider evaluating whether it should be retired.

Filter Form

Search Field (q=)
Text Input
Filters results by SKU, asset ID, or item name. The search is submitted by clicking "Apply" — it is NOT a live search. The page reloads with results.
Suggestion: Page-reload searches feel dated. Modern expectation is live filtering as you type. At minimum, pressing Enter should submit the form without requiring the user to click Apply.
Zone Dropdown
Dropdown
Filters by zone: All, "rfid", or "bin". Selecting a zone limits which inventory rows are shown. Submitted along with the search when "Apply" is clicked.
Suggestion: The zone names "rfid" and "bin" are internal codes that may confuse staff. Labels like "RFID Floor" and "Physical Bin" would be clearer.
Apply Button
Submit Button
Submits the filter form and reloads the page with filtered results.
Suggestion: Standard form submit. Works correctly. The button should be disabled and show a loading state while results are loading to prevent double-clicks.
Clear Link
Link
A link that reloads the page with no filters applied, resetting the view to show all items.
Suggestion: Works as expected. Adding an "×" icon to make it look like a clear button would make it more intuitive.
"View exceptions" Link
Link
A link in the top-right of the filter row that jumps to the Alerts & Exceptions page.
Suggestion: Good contextual shortcut. Consider showing the count of open exceptions next to this link (e.g., "View exceptions (3)") so users know if there is anything to act on.

Inventory Table

Results Table
Data Table
Shows up to 80 rows with columns: Asset (EPC-style ID), SKU, Name, Zone, and Quantity. The Asset ID is highlighted in teal. Rows highlight on hover. No way to sort by column. No way to click a row for more detail. No pagination — hard limit of 80 rows.
Suggestion: The 80-row hard limit is arbitrary and not communicated to users. A user seeing 80 results might not know there are more. Add a message like "Showing first 80 results — use search to narrow down." Also, column headers should be clickable for sorting. And clicking a row should open a detail view rather than doing nothing.
Empty State Message
Empty State
When no data is found, shows: "No rows — run npm run db:seed." This is a developer-only instruction that makes no sense to a warehouse operator.
Suggestion: Replace the empty state message with something user-friendly: "No inventory found. Try a different search term, or contact your administrator if you believe items should appear here." Remove the developer-facing database seed instruction entirely from the production build.
