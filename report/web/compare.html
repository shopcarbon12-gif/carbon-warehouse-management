RFID ↔ POS Compare (Developer Page)

A developer/testing utility page for running compare operations using a hardcoded sample payload. Not intended for daily warehouse use — the real compare tool lives in Reports → POS Compare.

Recommendations
⚠️ This page appears to be a development/testing tool. It uses hardcoded sample data and should not be accessible to regular warehouse staff.

Page Content

Cross-links to Real Pages
Navigation Links
Two links near the top: "Reports → POS compare" (the real compare workspace) and "Operations exceptions" (where generated exceptions can be reviewed). These point users to the correct pages.
The links to the real compare page and the exceptions page are helpful — someone who lands here accidentally can immediately find where they should actually be.
Sample Payload Preview
Code Display
Shows a raw JSON code block displaying hardcoded sample data that will be submitted when the button is clicked. Contains two sample items with known quantities for testing the compare API.
Showing raw JSON code to warehouse staff is not appropriate for a production UI. The only users who need to understand this are developers testing the API — this page should never be in the hands of warehouse operators.
Restrict this page to admin/developer roles only via role-based access control, or remove it from the sidebar entirely. If kept, add a large banner: "DEVELOPER TESTING TOOL — Not for production warehouse use."
Run Compare Button
Primary Button
Labeled "Run compare (POST /api/compare/runs)". Clicking submits the hardcoded sample data to the API, creating a compare run in the database and generating exceptions for any discrepancies. Shows "Running…" while in progress. After completion shows the run ID and exception count.
The button label exposes the internal API endpoint path ("/api/compare/runs") which is technical and confusing. More critically, clicking this in production creates fake compare runs and fake exceptions in the live system — requiring manual cleanup.
Repeatedly clicking this button creates multiple fake compare runs that pollute the exception history and activity log, potentially masking real issues.
Either remove this button in production (via an environment flag) or add a confirmation dialog: "This will create a test compare run using fake data in the LIVE system, generating fake exceptions. This is for developer testing only. Continue?"

Overall Assessment

This page should not exist in a production warehouse environment. It uses hardcoded fake data, exposes raw API paths, and creates real database records when clicked. The correct compare tool for warehouse use is Reports → POS Compare.
Recommendation: Remove this page from the sidebar navigation in production. Consider making it available only via a hidden developer URL, or add environment-based protection so it only renders in non-production environments. Until it's gated, any warehouse staff member who finds it can inadvertently create fake exceptions in the live system.
