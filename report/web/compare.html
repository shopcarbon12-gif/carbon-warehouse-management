RFID ↔ POS Compare (Developer Page)

A developer/testing utility page for running compare operations using a hardcoded sample payload. Not intended for daily warehouse use — the real compare tool lives in Reports → POS Compare.

⚠️ This page appears to be a development/testing tool. It uses hardcoded sample data and is described as a way to "test" the compare API. It should likely not be accessible to regular warehouse staff.

Page Content

Sample Payload Preview
Code Display
Shows a raw JSON code block displaying the hardcoded sample data that will be submitted when the button is clicked. Contains two sample items: "AVA MINI DRESS BLACK S" (2 in RFID, 2 in POS — matching) and "ISLA KNIT BLACK OS" (24 in RFID, 0 in POS — discrepancy).
Suggestion: Showing raw JSON to warehouse staff is not appropriate. This page should either be restricted to developers/admins only, or completely removed and replaced with the proper POS Compare report which uses real data.
Run Compare Button
Primary Button
Labeled "Run compare (POST /api/compare/runs)". Clicking submits the hardcoded sample data to the API, creating a compare run in the database and generating exceptions for any discrepancies found. Shows "Running…" while in progress. After completion shows the run ID and how many exceptions were created.
Suggestion: The button label exposes the internal API endpoint path ("/api/compare/runs") which is technical and confusing for regular users. Also, repeatedly clicking this in production creates fake compare runs and fake exceptions in the real database. This page should be protected or removed from production.
Cross-links to Real Pages
Links
Two links near the top: "Reports → POS compare" (the real compare workspace) and "Operations exceptions" (where generated exceptions can be reviewed). These point users to the correct pages.
Suggestion: Good that these links are here. Consider making this entire page redirect to "Reports → POS Compare" and eliminating the demo functionality in production.
