Inventory Adjustments Report

Records of quantity or location adjustments made to inventory — both automatic adjustments from system operations and manual corrections by staff. Essential for accountability and reconciliation.

Adjustments Table

Adjustments Log
Data Table
Shows inventory adjustment events with EPC or SKU, type of adjustment, quantity delta (positive = added, negative = removed), reason/source, and timestamp. Uses the inventory audit log workspace component.
Suggestion: Show the net impact at the top: "Total adjustments today: +47 items added, -12 items removed, net +35." Also add filtering by adjustment type (manual vs. automatic) and date range. Add CSV export.

Overall Adjustments Suggestions

1. Approval workflow: Large manual adjustments (e.g., removing 50+ items) should require a second approver. This prevents accidental or fraudulent changes going unnoticed.

2. Reason codes: Adjustments should have standardized reason codes (Received, Damaged, Sold, Count Correction, Theft) so patterns can be analyzed over time.
