Navigation & Sidebar

This sidebar wraps every page in the web app. It is the main way users move around the system. It appears on the left side of the screen on desktop and slides in as a drawer on mobile.

Top of the Sidebar

CarbonWMS Logo / Title Link
Clickable Link
Clicking the "CarbonWMS" name or the "WMS" badge at the top-left of the sidebar takes the user back to the Dashboard. It shows the branding text "WMS", then "CarbonWMS", then "RFID operations" underneath.
Suggestion: The subtitle "RFID operations" is very generic. Consider showing the currently active location name here (e.g. "Warehouse A") so the user always knows at a glance which location they are working in.
Location Switcher
Dropdown
Sits right below the logo. Lets the user switch between warehouse locations they have access to. The currently active location affects all data shown throughout the system — inventory counts, exceptions, transfers, etc.
Suggestion: When the user switches locations, there is no confirmation or visual flash to indicate the switch happened and all data has refreshed. Adding a brief toast message like "Switched to Warehouse B" would prevent confusion. Also consider showing a color-coded badge per location to make them visually distinct.
Close Sidebar Button (Mobile only)
Button
A small X button appears in the top-right corner of the sidebar on mobile. Tapping it closes the sidebar panel. On desktop this button is hidden since the sidebar is always visible.
Suggestion: The X button is small (only the icon, no label). On devices with thick fingers or gloves (warehouse workers often wear gloves), this could be hard to tap. Consider making the tap target larger — at least 44×44 pixels.

Dashboard Link (Top)

Dashboard
Navigation Link
A standalone link at the very top of the navigation list (above all the accordion groups). Clicking it goes to the Dashboard. When you are on the Dashboard, this link becomes highlighted in the accent color with a ring around it.
Suggestion: The Dashboard is the only item that sits outside of an accordion group, making it easy to find. Good. However, it would be helpful to show a notification badge here if there are unresolved exceptions, so users can see at a glance that something needs attention from any page.

Accordion Navigation Groups

The sidebar has 5 collapsible accordion sections. Clicking the section header expands or collapses it. When you are on a page inside a section, that section automatically expands.

Inventory (section)
Accordion Group
Contains: Catalog, Locations & Bins, Cycle Counts, Transfer Out, Transfer In, Bulk Status, and "Transfers (legacy)". Expands when you are on any of those pages. The chevron arrow rotates 180° when open.
Suggestion: The item "Transfers (legacy)" is labeled as legacy inside a modern section, which signals it probably should not be here long-term. Consider either removing it or adding a small deprecation warning when users click it, directing them to the newer Transfer In/Out pages.
RFID & Hardware (section)
Accordion Group
Contains: EPC Tracker, Print/Commission, Exceptions, and Devices. The "Exceptions" item has a red pulsing dot badge on its icon, meaning there are items that need attention. This badge is always visible when this section is open — it does not show a count, just a permanent red dot.
Suggestion: The red dot on Exceptions is always on — it never goes away, even if all exceptions are resolved. This creates "alert fatigue" where users start ignoring it because it never changes. The badge should dynamically reflect the actual count of open exceptions (e.g., "3") and disappear entirely when there are none.
Reports (section)
Accordion Group
Contains 9 report links: POS Compare, Device Upload Logs, Activity History, Asset Movements, Status & Tag Logs, Inventory Adjustments, Replenishments, Bulk Imports, External Systems.
Suggestion: Nine reports in one flat list can be overwhelming. Consider grouping them visually — e.g., "Inventory Reports" vs "System/Audit Reports". Also, the labels like "POS compare" could be clearer — something like "POS vs RFID Stock Compare" is instantly understandable by a warehouse manager.
Integrations (section)
Accordion Group
Contains two links: "Lightspeed Sync" and "LS Sales". Both are related to the Lightspeed POS integration.
Suggestion: With only 2 items, this section could be merged into another group (perhaps Settings or Infrastructure) to reduce the number of accordion groups the user has to navigate. Or, as the system grows with more integrations, this will make more sense.
Settings (section)
Accordion Group
Contains 10 items: General Settings, Theme & Style, Handheld Settings, Mobile OTA, Device Binding, Status Labels, RFID EPC General, EPC Profiles, Users & Roles, Locations. It expands when you are on any settings page.
Suggestion: Ten items is a lot for one section. Some items like "RFID EPC General" and "EPC Profiles" are very technical and would confuse most staff users. Consider hiding these behind a "Super Admin only" sub-group, or only showing them when the user's role has access. This would make the Settings section much less intimidating for regular users.

Bottom of the Sidebar

Sign Out Button
Button (Form Submit)
A small teal "Sign out" text link at the very bottom of the sidebar. Clicking it submits a server action that clears the session and logs the user out. There is no confirmation dialog — it signs you out immediately.
Suggestion: With no confirmation, a user could accidentally sign out if they click the wrong thing while scrolling. A simple "Are you sure?" confirmation (or an undo grace period) would prevent frustration. Also, it's easy to miss — it looks the same as a regular text link rather than a distinct button with a logout icon.

Mobile Behavior

Mobile Overlay & Backdrop
Overlay
On mobile, the sidebar slides in from the left and a dark transparent backdrop covers the rest of the screen. Clicking/tapping the backdrop closes the sidebar. This is standard mobile drawer behavior and works correctly.
Suggestion: The sidebar closes automatically when you tap a navigation link, which is good. However, there is no way to open the sidebar from within the content area — a hamburger menu button in the top header bar would make it easier to open the sidebar without having to swipe from the edge.

Overall Sidebar Suggestions

1. User info at bottom: There is no display of which user is currently logged in. Adding the user's name or email near the Sign Out button would help users confirm they are in the right account.

2. Active location more prominent: The Location Switcher is easy to miss. The current location should be displayed more clearly — possibly in the site header or as a persistent badge so users always know which warehouse data they are viewing.

3. Search/quick jump: With many pages across 5 accordion groups, adding a search box at the top of the sidebar (like pressing "/" to filter nav items) would save time for power users.

4. Role-based visibility: All navigation items are visible to all users regardless of role. Items the user cannot access (due to permissions) should either be hidden or shown greyed out with a lock icon, rather than showing the page but restricting it after the click.
