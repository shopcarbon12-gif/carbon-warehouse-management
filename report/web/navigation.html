Header & Branding

Logo / Home Link
Clickable Logo
The CarbonWMS logo at the top of the sidebar. Clicking it navigates to the dashboard.
Logo-as-home-button is a universal web convention — every user instinctively knows to click the logo to get back to the start.

Location Switcher

Location Switcher Dropdown
Dropdown
A dropdown at the top of the sidebar that lets the user switch between the warehouse locations they have access to. The currently selected location determines what inventory data, device scans, and reports are shown across the entire app. Changing the location reloads the relevant data.
A persistent location context visible in the sidebar means the user always knows which site they are viewing. Switching is fast and applies globally.
When a user switches locations, there is no visual confirmation that the switch happened and that the page data has updated. A brief "Now viewing: Main Warehouse" toast or highlight would prevent confusion when switching quickly.
After switching, show a brief flash/highlight on the location name and a small toast notification. Also consider remembering the last selected location between sessions so the user doesn't have to re-select on every login.

Navigation Accordion Sections

Accordion Groups (5 sections)
Collapsible Navigation
Five collapsible groups: Inventory, RFID & Hardware, Reports, Integrations, Settings. Each expands to show its sub-pages. Only the section containing the current page is open by default. Clicking a section header opens or closes it.
Accordion navigation keeps the sidebar compact. Showing only the current section open reduces visual noise — a manager in Reports isn't distracted by the full Settings list.
All five sections are shown to every user regardless of their role. A receiving staff member who can't access Settings still sees the Settings group. This creates confusion about what they are allowed to do, and makes the sidebar longer than it needs to be for most users.
Hide navigation sections the logged-in user does not have access to based on their role. A role with no access to Settings should not see the Settings section at all. This also serves as a security signal — visible links invite curiosity clicks.
Active Page Indicator
Visual State
The currently active page link is highlighted in the sidebar to show which page the user is on.
Clear active state highlighting is a basic but important UX requirement — users should always know where they are in the app. This is implemented correctly.

Sign Out

Sign Out Button
Action Button
Located at the very bottom of the sidebar. Submits a server action that clears the session and redirects to the login page. No confirmation dialog is shown.
No confirmation before signing out means an accidental click on Sign Out (which is easy when scrolling the sidebar) immediately ends the session. Any work in progress — an unsaved form, a half-completed cycle count — is lost without warning.
Add a single confirmation: "Sign out? Any unsaved changes will be lost." One extra click is a very small price for preventing accidental logouts during active work.

Overall Assessment

The sidebar works: accordion structure, logo link, active page highlighting, and location switcher are all solid foundational choices for a warehouse management app.
Role-based visibility is missing. Every user sees every navigation section. This is a security concern (curious staff can discover pages they shouldn't know about) and a UX problem (sidebar is cluttered for most users).
Priority fix: Hide nav sections based on user role. This single change cleans up the sidebar for all users and reduces accidental access to admin-only areas.
