Device Upload Logs

A report showing every batch upload that handheld devices have sent to the server. Each time a handheld completes a scan session and syncs, a record appears here with the device ID, batch ID, number of EPCs uploaded, and timestamps.

Upload Logs Table

Upload Logs Table
Data Table
Shows all handheld batch uploads. Typical columns include: Device identifier, Batch ID, Location, EPC count, and upload timestamp. Data is read-only. The workspace component handles loading and display.
Suggestion: Add filtering by device (so you can see all uploads from one specific handheld), by date range, and by location. Also add a CSV export option so this data can be shared in reports or audits. Consider showing whether each batch was processed successfully or had errors.
Empty/Loading States
States
Shows a loading indicator while data is being fetched. Shows an empty state message if no uploads have been recorded.
Suggestion: The empty state should explain what triggers uploads: "No uploads recorded yet. Upload logs appear here when handhelds sync their scan sessions to the server."

Overall Upload Logs Suggestions

1. Device health monitoring: This report could double as a device health dashboard — if a device hasn't uploaded in 24 hours, flag it as potentially offline or not being used.

2. Link to device registry: Each device ID shown should be a clickable link to that device's entry in the device registry (Infrastructure → Devices).

3. EPC drill-down: Clicking a row should expand it to show the individual EPCs that were uploaded in that batch.
