Integrations Table

Provider & Status Columns
Table Columns
Provider: shows the connected system name (e.g., "lightspeed"), auto-capitalized. Status: shows raw text — "active", "error", "pending" — with no color coding.
Status values are plain text with no color coding. An "error" status looks the same as "active" at a glance — an admin checking connection health has to read each cell carefully rather than scanning visually.
Color-code the status pill: green badge for "active," red for "error," amber for "pending." Consider showing a provider logo icon alongside the name for quick visual recognition.
Location Column
Table Column
Shows which warehouse location the integration is linked to (e.g., "STORE-A"). If the integration is at the tenant level (not location-specific), it shows "tenant."
"tenant" is an internal technical term. Warehouse admins using this page will not understand what "tenant" scope means in this context.
Replace "tenant" with "All locations" to make it immediately clear this integration applies across the entire warehouse system.
Last OK & Last Job Columns
Table Columns
Last OK: timestamp of the last successful API response. Last Job: timestamp of the last sync job run (not necessarily successful). Both show "—" if not available.
Both columns show raw ISO timestamps. Admins checking connection health need to mentally calculate how long ago each event was — "2026-04-09T10:30:00" doesn't immediately read as "2 days ago."
There is no "Last job result" column — you can see a job ran, but not whether it succeeded or failed. This makes diagnosing issues impossible from this page alone.
Show relative time ("2 hours ago") alongside the timestamp. Add a "Last job result" column (Success/Failure). Highlight "Last OK" in amber if it was more than 24 hours ago — this signals a potentially stale connection.
Empty State
Empty State
When no integrations are configured, shows: "No integration rows — connect Lightspeed or seed data."
"seed data" is developer language with no meaning to an admin. This message exposes the development process rather than guiding the user.
Replace with: "No integrations configured yet. Go to Settings → General to set up your Lightspeed connection." Include a direct button link to Settings → General.

Page Actions

Read-Only Page (No Actions)
Page Capability
This page is entirely read-only. There are no actions — no "Test connection," no "Force sync," no "Disconnect." Admins can only observe the status, then navigate elsewhere to act on it.
A read-only integrations page forces admins to navigate to Settings just to test or force-sync a connection. Every time an integration breaks, the diagnostic flow is: check this page → navigate to Settings → re-authorize → come back to verify. This is too many steps.
Add action buttons per integration row: "Test connection" (verify the credentials are still valid), "Sync now" (force an immediate sync), and "View sync history" (link to the External Systems report filtered to that provider). This transforms the page from a read-only display into a useful control panel.

Overall Assessment

The Integrations page shows data but cannot act on any of it. For admins monitoring integration health, the lack of color-coded status, relative timestamps, job results, and any action buttons makes this page nearly useless compared to what it could be.
Key improvements:
1. Color-code status: green = active, red = error, amber = pending.
2. Show relative time ("2 hours ago") for sync timestamps.
3. Add a "Last job result" column (Success/Failure).
4. Add action buttons: Test connection, Sync now, View history.
5. Replace "tenant" with "All locations" and remove developer empty state text.
