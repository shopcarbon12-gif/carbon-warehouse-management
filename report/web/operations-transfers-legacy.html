Operations — Transfers (Legacy)

An RFID-driven transfer tool where you select a destination and then scan items via a live RFID stream. Items are staged (reviewed) before being committed as a transfer. This is the older transfer workflow — consider using Transfer In/Out instead.

Destination Selection

Destination Location Dropdown
Dropdown
Populated from the locations registry. Selecting a location enables the bin dropdown for that location. Must be selected before scanning.
Only the destination is specified — the source (where items are being moved FROM) is never explicitly recorded. The source is inferred from each item's current location at scan time, which is implicit and cannot be verified by the person creating the transfer.
Capture the source location explicitly, or at minimum show a confirmation step that summarizes: "Moving items from [source locations] → [destination]." This makes the transfer record complete and verifiable.
Destination Bin Dropdown
Dropdown
Populated after selecting a location. Shows bins with their in-stock counts. Optional — you can commit a transfer to a location without specifying a bin.
Making bin selection optional allows inventory to be deposited at a location level without bin-level precision. Items that arrive with no bin assignment float as "location-level" stock, breaking cycle count accuracy.
Either make bin selection required, or show a clear warning when proceeding without a bin: "No bin selected — items will be assigned to [Location] without a bin. This reduces inventory precision."

Scanning

Live RFID Scan Stream
Live Stream
Connects to a real-time RFID event stream. When fixed readers are scanning and the scan context is set to "TRANSFER", EPCs are automatically added to the staging area. Avoids manual EPC entry.
Using the live RFID stream for transfers is efficient — items are added to the transfer list as they pass through the reader zone without manual entry.
The scan context must be manually set to "TRANSFER" for this to work. Floor staff should not need to configure scan contexts — this is a technical operation that can silently break the workflow if forgotten.
Set the scan context to "TRANSFER" automatically when this page opens, and reset it automatically when the page is closed or the transfer is committed. The context management should be invisible to the user.
Manual EPC Entry Field
Text Input
A field to manually type or paste an EPC code, as an alternative to live scanning. Useful when the reader is not available or for adding specific known tags.
The manual entry field has no explicit "Add" button shown. Pressing Enter should add the EPC — if this is not implemented, the field is just a display input with no obvious action.
Add a visible "Add" button next to the field. Pressing Enter after typing should also add the EPC. Show validation immediately if the EPC is not found in the system.

Staged Items & Commit

Staged Items List
Table
Shows all EPCs scanned or added manually, with their current location, bin, and status. This is the "review before commit" step.
There is no way to remove individual items from the staged list before committing — all staged items are committed together. A mis-scan cannot be corrected without discarding the entire session.
Allow removing individual items from the staged list with a trash icon per row. Show item names/SKUs alongside EPCs so staff can visually verify they have the right items before committing.
Commit Transfer Button → Modal
Button → Confirmation Modal
Opens a confirmation modal summarizing what will be transferred. Confirming commits all staged EPCs to the selected destination, updating their location in the database and writing an audit record.
The commit modal should clearly state: "Moving X items from [source locations] to Bin A4, Warehouse B. Items will immediately appear as in-stock at the destination." Make the impact explicit so the operator can verify before confirming.

Overall Assessment

This is a legacy page being replaced by the newer Transfer In/Out workflow. The source-location gap and the requirement to manually configure scan contexts are fundamental design issues. The newer Transfer In/Out pages address these properly with numbered slips and clear source/destination records.
Recommendation: Consider removing this page from the sidebar navigation and redirecting users to the Transfer Out page. If it must remain, add a clear notice at the top explaining when to use this vs Transfer In/Out, and add the ability to remove individual staged items before committing.
