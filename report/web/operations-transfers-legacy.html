Operations — Transfers (Legacy)

An RFID-driven transfer tool where you select a destination and then scan items via a live RFID stream. Items are staged (reviewed) before being committed as a transfer. This is the older transfer workflow — consider using Transfer In/Out instead.

⚠️ This page is labeled "Legacy" in the sidebar navigation. It may be replaced by the newer Transfer In/Out pages.

Destination Selection

Destination Location Dropdown
Dropdown
Populated from the locations registry. Selecting a location then enables the bin dropdown for that location. Must be selected before scanning can be meaningful.
Suggestion: The current location (where items are being moved FROM) is never specified in this workflow — only the destination. This makes the transfer history incomplete. The source should be captured automatically based on the item's current location when scanned.
Destination Bin Dropdown
Dropdown
Populated after selecting a location. Shows bins with their in-stock counts. Optional — you can commit a transfer to a location without specifying a specific bin.
Suggestion: Making bin selection optional could lead to inventory being deposited at a location-level without bin precision. Consider making bin selection required, or at minimum showing a warning if no bin is selected.

Scanning

Live RFID Scan Stream
Live Stream
The page connects to a real-time RFID event stream. When fixed readers are scanning and the scan context is set to "TRANSFER", EPCs are automatically added to the staging area. This avoids manual EPC entry.
Suggestion: The requirement to have the scan context set to "TRANSFER" is a technical detail that floor workers should not need to manage manually. This context should be set automatically when a transfer workflow is open, and reset when the page is closed.
Manual EPC Entry Field
Text Input
A field to manually type or paste an EPC code, as an alternative to live scanning. Useful when the reader is not available or for adding specific known tags.
Suggestion: Add an "Add" button next to this field. Pressing Enter after typing should also add the EPC to the list.

Staged Items

Staged Items List
Table
Shows all EPCs that have been scanned or added manually, along with their current location, bin, and status. This is the "review before commit" step — the user can see what they are about to transfer.
Suggestion: Allow removing individual items from the staged list before committing. Currently it appears all staged items are committed together. Also, show item names/SKUs alongside EPCs so staff can visually verify they have the right items.
Commit Transfer Button → Modal
Button → Confirmation Modal
Opens a confirmation modal summarizing what will be transferred. Confirming commits all staged EPCs to the selected destination location and bin, updating their location in the database and writing an audit record.
Suggestion: The commit modal should show a clear summary: "Moving 15 items from [various sources] to Bin A4, Warehouse B." Also clarify whether this updates inventory immediately or is queued for a worker process.
