🌐 Web App

Theme & Style

Lets each user pick their preferred visual style for the web app. Four preset combinations of color mode and font size. The preference is saved in the user's browser (not their account), so it needs to be set again on a different computer.

Theme Preset Cards

Dark · Default Type
Preset Card Button
Dark background with standard font size (~18px root). The former default "large" scale. When selected, shows a teal accent ring around the card and active styling.
Suggestion: The description "matches the former 'large' scale" implies there was a size change at some point. Remove historical notes from the UI — just describe the current state: "Dark background · Standard text size."
Dark · Extra Large Type
Preset Card Button
Dark background with larger font size. Description: "Larger root + roomier fields." The extra large scale makes form fields slightly taller too. Useful for users who find the default text too small.
Suggestion: This is a great accessibility feature. Consider adding a third intermediate size ("Compact") for power users who prefer a denser layout to see more data on screen at once.
Light · Default Type
Preset Card Button
Light/white background with standard font size. Useful for bright environments where a dark screen is harder to read.
Suggestion: In bright warehouse environments with fluorescent lighting, a light theme is often easier to read on desktop screens. This is a practical option that should be highlighted as suitable for day shift office use.
Light · Extra Large Type
Preset Card Button
Light background with larger fonts. Combines both accessibility features.
Suggestion: Good combination for accessibility. Each card should include a small preview thumbnail so users can see what the theme looks like before selecting it.

Overall Theme Settings Suggestions

1. Browser-only storage is limiting: The theme preference is saved in the browser (localStorage), not in the user's account. Warehouse staff often switch between computers (desktop at receiving, laptop at the office). Saving the preference server-side per user would fix this.

2. Live preview: Clicking a card should immediately change the app's appearance without needing to navigate away. This already works (the theme provider applies immediately), which is great.

3. System preference detection: Add a 5th option: "Use system setting" that automatically switches between dark/light based on the OS preference. This is now standard in modern apps.
