Print / Commission Labels

The tag printing workflow. Search the product catalog, configure how many labels to print and to which bin, review the label preview on a canvas, and send to the connected label printer. Status bar shows each stage: IDLE → GENERATING → PRINTING → SETTLE → DONE. Includes a print logs modal for history.

Product Search & Selection

Product Search
Search Input
A search field to find a product from the Lightspeed catalog by SKU or name. Results appear in a dropdown list. Selecting a product sets it as the item to be tagged.
Searching the catalog directly from this page means the product is correctly linked to the tag at the point of encoding — no separate lookup step required.
After selecting a product, the user sees the SKU and name but no stock level. Someone printing 10 labels for an item that already has 40 tagged copies may be creating duplicates unknowingly.
Show the current in-stock RFID count next to the selected product: "Current RFID tags: 12 in-stock." This lets the operator decide whether they actually need to print more labels.

Print Configuration

Quantity, Bin, Add-to-Inventory Settings
Settings Panel
Options: how many labels to print (quantity), which bin to assign the tagged items to, and whether to add the new items to the inventory automatically after printing. These settings persist between print jobs in the same session.
The "add to inventory automatically" toggle is a smart option — it removes a separate step for the most common workflow (print then receive).
Settings persist between print jobs within a session but are forgotten on page reload. If someone refreshes mid-job, they may start printing with wrong settings and not notice until labels come out wrong.
Save the last-used bin and quantity to localStorage so they survive page refreshes. Show a visible reminder of the current settings before each print job: "Printing 5 labels → Bin A3 → Auto-add to inventory."

Label Preview & Print Status

Label Preview Canvas
Visual Preview
A canvas element that renders a visual preview of the label that will be printed — showing the EPC barcode, QR code, and product text exactly as it will appear on the physical label.
Label preview before printing is excellent — it prevents misprints and saves label stock. Seeing exactly what will be printed builds confidence in the workflow.
Status Bar (IDLE → GENERATING → PRINTING → SETTLE → DONE)
Progress Indicator
A status bar with five stages that lights up as the print job progresses. Each stage has a distinct state. The bar provides real-time feedback on where the job is in the process.
Multi-stage progress is far better than a single spinner — operators know whether the delay is in tag generation, label formatting, or printer communication. This directly helps diagnose where a stuck job is failing.
If a stage fails (e.g., printer is unreachable), the bar stops at that stage but the error message appears below, away from the visual status indicator. The connection between "stuck at PRINTING" and "printer offline" requires reading two separate UI areas.
Show the error message directly under the stuck stage indicator: the PRINTING stage block turns red with "Printer unreachable — check network connection and printer power." This creates an immediate, clear connection between the failure point and its cause.
Print Logs Modal
History Modal
A button that opens a modal showing the history of all print jobs: product, quantity, timestamp, and success/failure status. Useful for auditing and troubleshooting.
Having an accessible print history without leaving the page is the right design. Operators can reference recent jobs without navigating to a separate Reports page.
The print logs are session-only or limited in history depth — long-term commissioning history requires going to the Reports section.
Add a direct "View full commissioning history →" link inside the modal that opens the relevant report page. This creates a natural escalation path from quick reference to deep audit.

Overall Assessment

The commissioning workflow is well-designed — catalog search, visual label preview, multi-stage progress, and print history are all present. The print preview canvas in particular is a feature that directly prevents waste and errors.
Key improvements:
1. Show current RFID stock count for the selected product to prevent over-tagging.
2. Inline error messages on the status bar stage that failed.
3. Persist settings across page reloads.
