Print / Commission (RFID Commissioning)

The most technically complex page in the system. Used to assign new RFID tags (EPCs) to products and send print jobs to RFID label printers. Involves searching the catalog, generating EPC codes, configuring printer settings, previewing labels, and tracking print history.

Product Search

Search Input (Product)
Search Field
Type to search for a product from the catalog. After a 280ms delay, results appear as a dropdown list. Requires at least 2 characters. Each result shows the SKU, item ID, description, and UPC. Clicking a result selects it and prepares the EPC generation form.
Suggestion: After selecting a product, it is not immediately obvious the product has been "selected" — consider highlighting the selected product with a clear confirmation panel. Also, support scanning a barcode directly into this search field (barcode scanners typically press Enter after scanning, so the system should handle that).

EPC Generation Settings

Quantity Field
Number Input
How many RFID tags to generate and print for the selected item. Defaults to 1. Each print job generates sequentially numbered EPCs starting from the next available serial number.
Suggestion: Add a max limit and a warning if the quantity seems unusually high. Also show "Next serial number: 147" so the user knows what the generated EPCs will look like before printing.
Bin Assignment Dropdown
Dropdown
A dropdown to select which bin the newly commissioned items should be placed in. Populated from the active location's bin list. This assigns the items to a bin at the moment of commissioning so they appear correctly in inventory reports.
Suggestion: If the active location has many bins, the dropdown can become very long. A searchable dropdown would be better. Also clarify what "no bin selected" means — does it put items in an unassigned state?
"Add to Inventory" Checkbox
Checkbox
When checked, the commissioned EPCs are immediately added to the inventory count at the selected location/bin. When unchecked, EPCs are generated and printed but not yet reflected in inventory (useful for staging or testing).
Suggestion: This is a critical checkbox — checking it has immediate real consequences for stock counts. It should be more visually prominent with a clearer label like "Add these items to inventory now" and perhaps an explanation of what happens if it is left unchecked.

Settings Panel

Settings Toggle / Panel
Expandable Panel
A settings panel that can be opened/closed. Contains printer configuration: printer IP/hostname, ZPL settings, label dimensions, EPC encoding parameters. Defaults are loaded from the system configuration.
Suggestion: These technical settings (ZPL printer language, EPC partition bits, etc.) are beyond most staff users. Consider having a simplified "use saved printer settings" mode that is the default, with an "Advanced settings" link for technicians only.

Label Preview

Preview Button & Canvas
Button / Canvas
Opens a label preview showing exactly how the printed label will look — including the EPC barcode, item name, SKU, and other fields laid out as they will appear on the physical label. Rendered as a canvas graphic.
Suggestion: This is excellent UX — previewing before printing saves wasted labels. Consider showing the preview automatically when all required fields are filled in, rather than requiring an extra button click.

Print / Commission Process

Status Bar
Status Display
Shows the current phase of the commissioning process: IDLE → GENERATING → PRINTING → SETTLE → DONE (or ERROR). Each phase has a visual indicator and timer. The "settle" phase waits 1.2 seconds after printing completes before allowing the next print job, to ensure the printer is ready.
Suggestion: The phase names (IDLE, SETTLE, etc.) are technical. For a warehouse operator, showing "Ready," "Sending to printer…," "Waiting for printer…," "Done — labels printed!" would be more natural. Also show the elapsed time more prominently.
Generated EPCs Display
Output
After a successful print job, the generated EPC codes are displayed. These are the unique identifiers encoded into the physical labels that were just printed. Each EPC follows the SGTIN-96 standard.
Suggestion: Show a "Copy all EPCs" button next to the generated list so they can be easily pasted into other tools (like the Bulk Status Update page). Also provide a way to download the list as a text file.
Error / Message Display
Status
If something goes wrong (printer unreachable, encoding failed, etc.), an error message appears. Success messages also appear here. Both types are styled differently.
Suggestion: Error messages for printer issues should include actionable guidance: "Could not reach printer at [IP address]. Check that the printer is powered on and connected to the network." Generic errors like "Print failed" without a reason are not helpful for floor staff.

Print Logs Modal

Print Logs Button / Modal
Button → Modal
Opens a modal/popup showing the history of print jobs on this page (within the current session). Shows what was printed, when, how many labels, and which EPCs were generated.
Suggestion: Print logs should persist across sessions and be available in the Reports section, not just while the page is open. If a user accidentally closes the tab after printing, they lose the record of what EPCs were generated.
