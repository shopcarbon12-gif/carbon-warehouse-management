Inventory Catalog

The main inventory view. Two tabs: Lightspeed (shows product lines synced from the POS — SKU, description, vendor, price) and RFID (shows individual scanned tags with their status and bin location). Search, pagination at 50 rows per page, CSV export, and admin-only sync controls.

Tab Navigation

Lightspeed Tab / RFID Tab
Tabs
Two tabs at the top. Lightspeed tab shows synced catalog lines from the POS system. RFID tab shows individual tagged items with their current status, bin, and EPC code. Switching tabs preserves the search query.
Separating the catalog view (what should exist) from the RFID view (what actually exists physically) is a clean distinction that matches how warehouse staff think about inventory.
The tab labels "Lightspeed" and "RFID" are technical platform names, not task-oriented language. A new employee doesn't know what "Lightspeed" means in this context.
Rename to "Product Catalog" and "Tagged Items" or "Physical Stock." Include item counts in the tab label: "Tagged Items (4,231)" so users know roughly how much data each tab contains.

Search

Search Field
Text Input (debounced)
A search field that triggers after the user stops typing (320ms debounce). Searches across SKU, product name, and EPC depending on the active tab. Results update without pressing Enter.
Debounced live search is the right pattern — avoids flooding the server on every keystroke while still feeling responsive.
There is no placeholder text explaining what can be searched. A user on the RFID tab doesn't know whether they can type a product name or only an EPC code.
Add a tab-specific placeholder: on Lightspeed tab — "Search by SKU, name, or vendor"; on RFID tab — "Search by EPC, SKU, or name." A clear prompt removes the guessing.

Data Table

Inventory Table (50 rows/page)
Data Table
Shows inventory rows paginated at 50 per page. Columns vary by tab — Lightspeed shows SKU, description, vendor, price, quantity. RFID shows EPC, SKU, status, bin, last-seen date. Clicking a row opens a detail modal.
Row-click to open detail is intuitive. The separate columns per tab are appropriate — the two data sets have different relevant attributes.
50 rows per page with no option to change is inflexible. Admins doing data reviews often want to see 100+ rows. There is also no column sorting — clicking a column header does nothing.
Add page size options (25 / 50 / 100). Make column headers sortable — especially "Status" and "Last Seen" on the RFID tab, which are the most commonly used filters during a discrepancy review.
Item Detail Modal
Modal
Clicking a row opens a modal showing full details for that item: all fields, EPC history, current bin assignment, and status history. Admins can trigger actions from here.
A full detail panel without leaving the table is the right design — users can quickly check an item and return to browsing without navigating away.
No keyboard navigation between items. Once a modal is open, the user must close it and click another row. With 50 rows per page, reviewing many items is slow.
Add previous/next arrow buttons in the modal to step through items one by one without closing and reopening. This is especially useful during a review session.

Actions

Export CSV Button
Secondary Button
Downloads a CSV of the currently visible inventory data. Exports all rows matching the current search filter, not just the current page.
Exporting all filtered rows (not just the visible page) is the correct behavior — users shouldn't have to page through the entire table to get a full export.
No feedback is shown during export — no spinner, no download started notification. For large exports, the button appears unresponsive.
Show a brief "Preparing download…" state on the button after clicking. For very large exports, consider a toast: "Export ready — check your Downloads folder."
Sync Buttons (Admin Only)
Admin Action Buttons
Buttons to manually trigger a Lightspeed catalog sync. Only visible to admin-role users. Pulls the latest product data from Lightspeed and updates the local catalog.
Hiding admin-only sync controls from regular staff is correct — prevents accidental triggering of a heavy background operation.
No indication of when the last sync ran or whether the current catalog data is fresh. A catalog that hasn't synced in 3 days may show outdated prices or missing products.
Show "Last synced: 2 hours ago" near the sync button. If the last sync was more than 24 hours ago, highlight this in amber as a warning.

Overall Assessment

The two-tab catalog is a strong design — separating POS products from physical RFID tags matches exactly how warehouse staff and managers think about the distinction between "what we ordered" and "what we actually have."
Missing column sorting on the RFID tab makes discrepancy reviews slow. Sorting by "Status" to see all damaged items together, or by "Last Seen" to find items not scanned recently, are the most common investigative patterns.
Top priorities:
1. Sortable columns on RFID tab (Status, Last Seen).
2. Better tab labels — "Product Catalog" and "Tagged Items."
3. Search placeholder text per tab.
4. "Last synced" timestamp near sync buttons.
