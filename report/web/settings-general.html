🌐 Web App

General Settings (Infrastructure Settings)

The main configuration page for the WMS. Organized into three tabs: RFID settings (reader defaults), Integrations (Lightspeed OAuth credentials and connection setup), and Bins (default bin configuration). All changes are saved to the database for the tenant.

RFID Tab

RFID Settings Fields
Form Fields
Contains RFID-specific defaults for this tenant: EPC filter value, read power settings, antenna configuration, and session parameters. These affect how all RFID readers at this tenant behave by default.
Suggestion: RFID settings like "filter value" and "session parameters" are very technical. Each field should have a plain-language tooltip or help text explaining what it controls and what happens if it is changed. Wrong settings can cause all readers to stop working correctly.

Integrations Tab (Lightspeed)

Lightspeed Readiness Panel
Status Panel
Shows whether the Lightspeed live catalog connection is ready. Green panel = all credentials are configured and a real sync can run. Amber panel = something is missing, lists exactly what needs to be configured. Missing items include: Client ID, Account ID, LS_CLIENT_SECRET (server environment variable), and LS_REFRESH_TOKEN.
Suggestion: This readiness panel is excellent UX — it clearly tells the user what is missing and what to do. The one improvement: the steps mention "Coolify → WMS → Environment" which is a specific deployment platform. This should be written generically: "In your server's environment variables" with Coolify-specific instructions shown only if Coolify is detected.
Client ID Field
Text Input
Allows saving the Lightspeed Client ID in the database (as an alternative to the environment variable). The saved value supplements or overrides the environment variable.
Suggestion: Storing OAuth client IDs in the database rather than environment variables has security implications. There should be a note explaining why one method is preferred over the other. Also, the field should mask the value after saving (like a password field) so credentials are not visible to anyone who opens the settings page.
OAuth Flow (Start OAuth Button)
Button
Initiates the Lightspeed OAuth authorization flow — opens Lightspeed's authorization page in the browser. The callback URL for the OAuth redirect is shown so it can be configured in Lightspeed's developer portal.
Suggestion: The OAuth flow involves multiple steps across two systems (CarbonWMS and Lightspeed). A step-by-step wizard guide (Step 1: Copy this callback URL → Step 2: Open Lightspeed developer portal → Step 3: Paste and authorize) would make this much less error-prone.

Bins Tab

Bin Default Settings
Form Fields
Configuration for default bin behavior — likely including default bin capacity, naming conventions, or zone assignments for new bins.
Suggestion: Bin defaults should have descriptive help text explaining how each default is applied — e.g., "Default capacity: used when creating a new bin without specifying capacity."

Overall General Settings Suggestions

1. Save button state: It should be very clear when unsaved changes exist — a persistent "Unsaved changes" banner or a highlighted Save button. Currently users might navigate away thinking they saved when they didn't.

2. Change history: Settings changes should be logged in the activity audit so administrators can see who changed what and when.

3. Dangerous settings warning: Some RFID settings, if changed incorrectly, will break all reader functionality. Changing these should show a warning: "Changing this value will affect all readers at this tenant. Changes take effect on the next reader sync."
