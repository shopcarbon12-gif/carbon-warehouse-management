1. Executive summary

Pic1 and Pic2 diverge on screen background, logo treatment, server URL label layout, device identifier typography, and—most visibly—email/password field structure (Pic1 shows a nested outlined inner input inside the gray tray; Pic2 describes a single flat tray per field without that inner chrome). The Flutter code today uses InputBorder.none and a uniform _loginTray; any inner border in Pic1 may come from the platform TextField theme, focus state, or OEM rendering—this must be verified on device when implementing fixes.

Logo conflict to resolve before build: Pic2’s written spec describes a rounded square with “CarbonWMS / logo” text. The product direction elsewhere uses the hexagon PNG asset. The HTML preview below shows the Pic2 text logo for mock fidelity only; implementation should follow whichever asset you approve after this report.

3. Implementation cross-reference (Flutter)

Current code lives in mobile/carbon_wms/lib/ui/screens/login_screen.dart:

  • _bg = 0xFFF5F5F5 → conflicts with Pic2 white.
  • _fieldFill = 0xFFECECEC → near Pic2; optional tweak to #EBEBEB.
  • _loginTray wraps email/password TextField with InputBorder.none — inner outline in Pic1 likely not from explicit border in this file.
  • Device line uses GoogleFonts.inter and _deviceLineGrey → change for Pic2 monospace black.
  • SSL + label: built as one Row → Pic2 wants SSL visually “above” the tray.
