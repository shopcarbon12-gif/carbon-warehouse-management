Default Mode

Every single time a worker enters this screen — whether they just logged in, came back from another screen, or were here before — the screen resets completely. Nothing from the previous session is remembered. The bin box is empty, the badge says INACTIVE, there are no stored items, and the only thing they can do is scan a bin.

void _resetToDefault() {
  setState(() {
    _currentBin     = '';          // empty bin
    _currentBinId   = null;        // no bin UUID
    _storedContents = [];           // no items
    _undoSnapshot   = null;        // no undo buffer
    _phase          = _PutawayPhase.scanBin;
    _bottomLabel    = 'TRIGGER TO ADD BIN';
    _labelIsRed     = true;
    _binConfirmed   = false;        // locks ADD ITEM
  });
}

@override
void initState() {
  super.initState();
  _resetToDefault();
}

Step 1 — Scan Bin

The worker picks up the handheld gun (or uses camera / Bluetooth scanner) and scans the barcode on the bin shelf. The raw scan comes in as something like 2B03L — the app always reformats it to 2-B-03-L before displaying or sending to the server. Then the app checks if that bin already exists in the WMS system.
Formatting rule
Raw scan is always 5 characters in the pattern X-X-XX-X. The app splits them with dashes automatically. 2B03L → 2-B-03-L
DART — Format bin code
String _formatBinCode(String raw) {
  final s = raw.replaceAll(RegExp(r'[^A-Za-z0-9]'), '').toUpperCase();
  if (s.length != 5) return s; // fallback — show as-is
  return '${s[0]}-${s[1]}-${s.substring(2, 4)}-${s[4]}';
  // 2B03L → 2-B-03-L
}
Flow

Worker scans bin barcode → raw value received

Hardware wedge fires hidden TextField onSubmitted, camera returns string, Bluetooth keyboard fires same TextField

Format raw → X-X-XX-X → display immediately in bin box

API call: GET /api/locations/bins → filter response by code

Bin found → badge = ACTIVE → load stored contents → label → "TRIGGER TO ADD ITEM" (red) → unlock ADD ITEM

Bin not found → popup: "This bin does not exist. Would you like to add it?"

Input pre-filled with formatted code. Confirm → POST new bin → proceed as ACTIVE. Cancel → close popup, stay on screen.

DART — Bin scan handler
Future<void> _onBinScanned(String raw) async {
  final code = _formatBinCode(raw);
  setState(() => _currentBin = code);

  final api   = context.read<WmsApiClient>();
  final bins  = await api.fetchBins();           // GET /api/locations/bins
  final found = bins.firstWhereOrNull(
    (b) => b['code'] == code
  );

  if (found != null) {
    await _confirmBin(found['id'], code);
  } else {
    _showNewBinDialog(code);
  }
}

Future<void> _confirmBin(String id, String code) async {
  final contents = await api.fetchBinContents(id); // GET /api/locations/bins/contents?binId=
  setState(() {
    _currentBinId   = id;
    _currentBin     = code;
    _binConfirmed   = true;
    _storedContents = contents;
    _bottomLabel    = 'TRIGGER TO ADD ITEM';
    _labelIsRed     = true;
  });
}
DART — Create new bin dialog
void _showNewBinDialog(String prefilledCode) {
  final ctrl = TextEditingController(text: prefilledCode);
  showDialog(
    context: context,
    builder: (_) => AlertDialog(
      title: const Text('Bin not found'),
      content: Column(children: [
        const Text('This bin does not exist. Would you like to add it?'),
        TextField(controller: ctrl),
      ]),
      actions: [
        TextButton(onPressed: () => Navigator.pop(context), child: const Text('Cancel')),
        FilledButton(
          onPressed: () async {
            Navigator.pop(context);
            final id = await api.createBin(ctrl.text); // POST /api/locations/bins
            await _confirmBin(id, ctrl.text);
          },
          child: const Text('Add Bin'),
        ),
      ],
    ),
  );
}

Step 2 — Scan Item

After the bin is confirmed, the worker scans the barcode on the item. The app reads the SKU, figures out which product and color it belongs to (by stripping the size off the end), and searches the WMS catalog. A popup asks to confirm the assignment.

DART — Parse SKU into parts
class SkuParts {
  final String base;        // 7 or 9 chars (incl C)
  final String baseColor;   // 9 or 11 chars — search key for specific color
  final String colorCode;   // ZZ — 2 chars
  final String size;         // remainder
  const SkuParts({
    required this.base,
    required this.baseColor,
    required this.colorCode,
    required this.size,
  });
}

SkuParts parseSku(String sku) {
  final s = sku.trim().toUpperCase();
  final hasC  = s.startsWith('C');
  final baseLen = hasC ? 9 : 7;   // base length
  final bcLen   = hasC ? 11 : 9;  // base+color length

  if (s.length < bcLen) {
    // Too short — treat entire string as base+color, no size
    return SkuParts(base: s.substring(0, baseLen.clamp(0, s.length)),
                    baseColor: s, colorCode: '', size: '');
  }

  return SkuParts(
    base:      s.substring(0, baseLen),
    baseColor: s.substring(0, bcLen),   // search key for specific color
    colorCode: s.substring(baseLen, bcLen),
    size:      s.substring(bcLen),
  );
}
DART — Item scan handler
Future<void> _onItemScanned(String raw) async {
  final parts   = parseSku(raw);
  setState(() => _busy = true);

  // Search catalog by base+color (no size)
  final result = await api.catalogGridSearchFirstRow(parts.baseColor);
  // GET /api/inventory/catalog?view=grid&q=112220507

  setState(() => _busy = false);
  if (result == null) { _showNotFoundSnack(); return; }

  _showAssignDialog(
    itemName:  result['description'] as String,
    skuParts:  parts,
    matrixId:  result['id'] as String,
  );
}
DART — Assign confirmation dialog
void _showAssignDialog({
  required String itemName,
  required SkuParts skuParts,
  required String matrixId,
}) {
  bool addAnother = false;
  showDialog(
    context: context,
    builder: (ctx) => StatefulBuilder(
      builder: (ctx, setSt) => AlertDialog(
        title: Text('Assign to bin?'),
        content: Column(mainAxisSize: MainAxisSize.min, children: [
          Text('Would you like to assign ${itemName} to this bin?'),
          CheckboxListTile(
            value: addAnother,
            onChanged: (v) => setSt(() => addAnother = v ?? false),
            title: Text('Add another item?'),
          ),
        ]),
        actions: [
          TextButton(onPressed: () => Navigator.pop(ctx), child: Text('No')),
          FilledButton(
            onPressed: () async {
              Navigator.pop(ctx);
              // Assign all sizes of this color to bin
              await api.postPutawayAssign(
                binCode: _currentBin,
                skuScanned: skuParts.baseColor,
                scope: 'single_color',   // base+color = one color, all sizes
              );
              await _refreshContents();
              if (addAnother) _showNextProductDialog(skuParts, matrixId);
            },
            child: Text('Yes'),
          ),
        ],
      ),
    ),
  );
}

Same Product Different Color Flow

If the worker checked "Add another item?" and chose "Same product different color", the app searches the full product matrix (base only, no color). It shows a list of all available colors that are not yet assigned to this bin. The worker checks one or more colors and taps YES. The app assigns all sizes of each selected color in the background, then shows the same question again.

Search base only: GET /api/inventory/catalog?view=grid&q=1122205

Get all custom SKUs for matrix: GET /api/inventory/catalog?matrixId=

Filter out colors already assigned to this bin

Show color list with checkboxes (multi-select)

YES → for each checked color: POST /api/inventory/putaway-assign with scope: single_color → refresh → show "same or new?" popup again

NO → go back to previous popup

DART — Assign multiple colors
Future<void> _assignColors(
  List<String> colorSkus,   // each is a baseColor key e.g. "112220507"
) async {
  for (final colorSku in colorSkus) {
    await api.postPutawayAssign(
      binCode:    _currentBin,
      skuScanned: colorSku,
      scope:      'single_color',
    );
  }
  await _refreshContents();
}

6

Stored Items Display

The list shows every SKU assigned to this bin. The counter at the top shows how many unique color groups are assigned (not the total number of physical items). Each row shows the item name, color, size, and how many physical RFID tags (EPCs) exist for that SKU. Rows with zero EPCs are hidden — they're still assigned, just not shown.
ℹ️
Quantities shown are EPC counts from RFID — not catalog quantities from Lightspeed. The app assigns SKU details only; physical counts come from scanned RFID tags.
⚠️
When a bin is scanned and confirmed, existing contents are loaded immediately. New assignments add to existing — never overwrite.
DART — Refresh bin contents
Future<void> _refreshContents() async {
  if (_currentBinId == null) return;
  final rows = await api.fetchBinContents(_currentBinId!);
  // GET /api/locations/bins/contents?binId=
  // Returns: [{ sku, description, color_code, size, qty (epc_count) }]

  setState(() {
    // Hide rows with 0 EPCs from display (still assigned in DB)
    _storedContents = rows.where((r) => (r['qty'] as int) > 0).toList();

    // Counter = unique base+color groups
    _uniqueColorCount = rows
      .map((r) => r['sku'] as String)
      .map((sku) => parseSku(sku).baseColor)
      .toSet()
      .length;

    _bottomLabel = 'READY FOR NEXT ENTRY';
    _labelIsRed  = false;
  });
}

Swipe to Delete

Swipe a row to the right and a red trash icon appears on the left. Tap it and a confirmation popup asks "Are you sure?". YES removes that specific SKU and all its EPCs from the bin — nothing is deleted from the catalog or anywhere else in the system.
🗑️
Delete from bin only. Catalog, inventory records, and EPC registrations are never touched.
DART — Dismissible row
Dismissible(
  key: ValueKey(row['sku']),
  direction: DismissDirection.startToEnd,   // swipe right
  confirmDismiss: (_) async {
    return await _confirmDelete(row['sku']);  // shows YES/NO dialog
  },
  onDismissed: (_) async {
    await api.removeSkuFromBin(
      binId: _currentBinId!,
      sku:   row['sku'],
    );
    await _refreshContents();
  },
  background: Container(
    color: Colors.red,
    alignment: Alignment.centerLeft,
    padding: EdgeInsets.only(left: 20),
    child: Icon(Icons.delete, color: Colors.white),
  ),
  child: _StoredItemRow(row: row),
)
9

Clean & Empty Bin

Tapping CLEAN & EMPTY BIN asks for confirmation first. If confirmed, all items are cleared from the bin and an UNDO button appears. Tapping UNDO restores everything exactly as it was before the clean. If all EPCs for all assigned SKUs are already 0, the bin can auto-empty itself.
DART — Clean bin with undo snapshot
Future<void> _onCleanBin() async {
  final confirmed = await _showConfirm('Are you sure you want to empty this bin?');
  if (!confirmed) return;

  // Save snapshot for undo
  _undoSnapshot = List.of(_storedContents);

  await api.postCleanBinByCode(_currentBin);
  // POST /api/mobile/clean-bin  { binCode }

  setState(() {
    _storedContents = [];
    _showUndo = true;
  });
}

Future<void> _onUndo() async {
  if (_undoSnapshot == null) return;
  // Re-assign each SKU back to bin
  for (final row in _undoSnapshot!) {
    await api.postPutawayAssign(
      binCode:    _currentBin,
      skuScanned: row['sku'],
      scope:      'single_color',
    );
  }
  await _refreshContents();
  setState(() { _showUndo = false; _undoSnapshot = null; });
}

Ready for Next Entry

After each successful assignment, the screen refreshes showing all stored items. The bin stays active. The label goes back to "READY FOR NEXT ENTRY" in normal color. The worker can tap ADD ITEM again to scan another item into the same bin.

Camera Behaviour

Camera opens on tap of the camera icon inside ADD BIN or ADD ITEM. The camera screen shows an instruction at the top so the worker knows what to scan. It works in both portrait and landscape — the device orientation lock is overridden just for the camera screen.
ButtonScreen overlay textActive when
ADD BIN cameraSCAN BIN LOCATIONNo bin assigned yet
ADD ITEM cameraSCAN ITEMBin confirmed
DART — Override orientation for camera screen only
Future<String?> _openCamera(String overlayLabel) async {
  // Unlock all orientations before opening camera
  await SystemChrome.setPreferredOrientations([
    DeviceOrientation.portraitUp,
    DeviceOrientation.landscapeLeft,
    DeviceOrientation.landscapeRight,
  ]);
  final result = await openCameraBarcodeScanner(context, title: overlayLabel);
  // Re-lock to portrait after camera closes
  await SystemChrome.setPreferredOrientations([DeviceOrientation.portraitUp]);
  return result;
}

Widget _locked(Widget child, {required bool locked}) {
  if (!locked) return child;
  return Opacity(
    opacity: 0.35,
    child: IgnorePointer(child: child),
  );
}
